# GHA タスク分割でのステップ名参照優先ルール

## 知見
GHA ワークフローのタスク分割（02a_task_division.md）で変更箇所を指定する際、行番号よりステップ名（`name:` 属性）やセクション名での参照を優先すべき。行番号はファイル更新で容易にズレるが、ステップ名は安定した参照ポイントになる。

## 発見元
gha-pipeline-autorun 施策 L2-plan-evaluator

## 適用先
l2-plan-worker エージェント定義（`agents/l2-plan-worker.md`）の「02a_task_division.md のフォーマット」セクションに「行番号は参考値として記載し、ステップ名/セクション名を主たる参照とする」旨のガイドを追加
