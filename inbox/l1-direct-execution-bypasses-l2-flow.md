# L1 が汎用サブエージェントで直接作業を実行し L2 フローを迂回した問題

## 発生日
2026-03-17

## 発生施策
`sessions/initiatives/session-lifecycle-policy-rename/`

## 何が起きたか
L1 マネージャーが L2-worker / L2-evaluator エージェントをディスパッチせず、Agent ツールの汎用サブエージェントで実作業（パス一括置換・TG-008 移動）を直接実行した。その結果、以下のファイルがテンプレートのまま残った:

- 03_work_log.md（作業履歴なし）
- 04_work_report.md（作業レポートなし・知見抽出なし）
- 05_eval_plan.md（評価計画なし）
- 06_eval_report.md（評価レポートなし）

## 影響
- **作業の追跡可能性の喪失**: 何をいつ行ったかの記録がなく、コミット差分からの事後復元が必要になった
- **知見抽出の欠落**: work_report / eval_report の知見セクションが空で、横展開可能な知見が失われた可能性がある
- **ゲート判定の品質低下**: 正規の判定材料（作業レポート・評価レポート）がなく、コミット差分ベースの事後判定になった
- **責務分離の違反**: CLAUDE.md の「L1セッションが work_log を直接編集すること」の禁止事項に準ずる問題（今回は編集すらしなかった）

## 根本原因
- 「単純リネーム」という作業性質から L1 が効率を優先し、L2 フローを省略可能と判断した
- L1 の SKILL.md に「汎用サブエージェントでの直接作業実行の禁止」が明文化されていない
- 02_tasks.md で L2-worker にディスパッチ計画を書いたにもかかわらず、実際にはディスパッチしなかった（計画と実行の乖離）

## 再発防止の提案
1. **L1 SKILL.md にガードレールを追加**: 「L1 は Agent ツールで実作業を直接実行してはならない。必ず L2-worker / L2-evaluator エージェントをディスパッチすること」を明文化
2. **小規模施策でも L2 フロー必須**: 施策の規模に関わらず、02_tasks.md にディスパッチ計画を書いた以上は L2 ディスパッチを実行する
3. **ゲート判定チェック項目の強化**: Phase B→C チェックで「04_work_report.md がテンプレートのままでないこと」を必須ブロッカーとする

## 重大度
**高** — プロセス改善リポジトリの根幹であるセッション構成（L1/L2 責務分離）が機能しなかった。知見喪失は取り返しがつかない。
