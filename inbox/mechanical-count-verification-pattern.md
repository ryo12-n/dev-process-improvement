# 大量エントリ分類作業での機械的カウント照合パターン

## 気づき
backlog-inventory-batch 施策で、テーブル実データとサマリ集計値の不整合が発生した。分類作業の途中で集計を取り、その後テーブルを更新した際にサマリを更新し忘れたものと推定。

## 提案
大量エントリの分類作業を含むタスクの完了条件に「最終コミット前に grep 等でテーブル内のラベルを機械的にカウントし、サマリと照合するステップ」を含める。
