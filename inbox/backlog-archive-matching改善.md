# backlog と archive のマッチング改善

## 背景

トリアージ 20260308 で `claude-directory最適化(ai-driven-dev-patterns)` と `_archive/claude-directory最適化/` の同一施策判定に手間取った。対象リポジトリの違いが施策名に含まれたり含まれなかったりするため、機械的な突合が困難。

## 改善案

1. **l1-manager クローズチェックリストに backlog.csv 削除ステップを追加** — 施策化時に backlog エントリを確実に削除/更新する（ISS-041 の提案と同方向）
2. **施策名の命名規約を強化** — 対象リポジトリが異なる同名施策がある場合、必ずサフィックス `(リポ名)` を付ける。archive 側のディレクトリ名にも同じサフィックスを使う
3. **backlog テンプレートに「対象リポジトリ」フィールドを必須化** — 現状は任意記載で漏れがある

案1（クローズ時の確実な削除）と案2（命名規約統一）の組み合わせが最も効果的。
