# セッション定義テンプレートジェネレータ（メタテンプレート）の導入検討

## 背景

session-consistency-manager 施策において、config-optimizer-manager を参照パターンとして3フェーズ構成のセッション定義を作成した。両者の構造的対称性が高く、CO-XXX → SC-XXX のターゲット名置換と固有ロジックの差し替えで大半のファイル（SKILL.md + 4エージェント + リファレンス + 15テンプレート = 21ファイル）を効率的に作成できた。

## 提案

将来的に同様の3フェーズ構成セッションを追加する際の効率化のため、メタテンプレート（セッション定義のジェネレータ）の導入を検討する。

## 期待効果

- 新セッション定義作成の工数削減
- セッション間の構造的整合性の自動担保
- 連動ファイル更新の漏れ防止
