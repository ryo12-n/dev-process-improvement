# サブエージェントのコミット・プッシュ信頼性改善

## 背景

l1-manager-enhanced-planning 施策（フェーズ1・2 の両方）で、全サブエージェント（l2-plan-worker, l2-worker, l2-evaluator）のコミット・プッシュが失敗する事象が継続的に発生した。

## 影響

- L1 マネージャーがサブエージェントの成果物を手動でコミット・プッシュする補完作業が常態化
- 評価レポート（06_eval_report.md）が未更新のまま L1 が評価結果を口頭確認で代替するケースが発生
- 施策実行の主要オーバーヘッドとなっている

## 提案

- サブエージェントのコミット失敗の根本原因を調査し、改善策を検討する
- 代替案: L1 がサブエージェント完了後に必ずコミット・プッシュを行うフローを manager-common-policy に明記する
