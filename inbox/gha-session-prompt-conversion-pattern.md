# GHA 自動セッション用プロンプト変換パターン

## 概要

対話型スキル定義（SKILL.md）を GHA 非対話プロンプトに変換する際の共通パターン。

## 変換ルール

1. **除外するステップ**: ブランチ整理、壁打ちフェーズ、ユーザー確認待ち、セッションライフサイクル todo 登録
2. **サブエージェント置換**: ワーカー・評価者のディスパッチを単一エージェントの逐次実行に置換
3. **コミット委譲**: コミット・プッシュ・PR 作成は GHA ワークフローステップに委譲し、プロンプトからは除外

## 適用実績

- `gha-triage-automation` 施策で `/triage-manager` SKILL.md → `.github/prompts/triage.md` に適用

## 今後の適用候補

- メタ認知セッション自動化
- バックログメンテナンス自動化
