# backlog エントリ起票時の記述粒度最低基準

## 気づき

BL-084〜BL-093 の新規エントリの記述が簡潔すぎ、BM-005（重複・統合候補検出）の精度に影響を与えている。特に「課題・背景」セクションが1-2行のエントリでは、他エントリとの類似度判定が困難。

## 提案

backlog/entries/_template.md に記述粒度の最低基準を追加する:
- 「課題・背景」セクション: 最低3行（何が問題か、なぜ問題か、どの範囲に影響するか）
- 「期待効果」セクション: 最低1行の具体的な効果記述
- トリアージワーカーの起票チェックリストに粒度チェックを追加
