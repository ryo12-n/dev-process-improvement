# 改善施策提案: setting.json へ allow リスト追加

## 背景・課題

Claude Code のセッション中、ファイル操作・git コマンド・GitHub CLI 等のよくある操作で毎回許可プロンプトが表示され、作業効率が低下していた。
`.claude/settings.json` には `deny` ルールのみが設定されており、`allow` ルールが未設定だった。

## 目標

- よくある安全な操作を事前許可し、許可プロンプトの頻度を大幅に削減する
- 機密ファイル（`.env`, `secrets/`, `*.pem`, `*.key`）の保護は維持する

## スコープ

### やること
- `.claude/settings.json` に `allow` ルールを追加
- ファイル操作（Read/Edit/Write/Glob/Grep）の全ファイル許可
- git、GitHub CLI の操作許可
- シェルの読み取り系・基本変更系コマンドの許可

### やらないこと
- `deny` ルールの変更（既存のまま維持）
- ユーザーグローバル設定（`~/.claude/settings.json`）の変更

## 備考

- `deny` は `allow` より優先されるため、保護対象ファイルへのアクセスは引き続きブロックされる
- PR #6 で対応
