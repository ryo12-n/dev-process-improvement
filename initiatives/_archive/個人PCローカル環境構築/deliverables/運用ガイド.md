# ローカル環境 運用ガイド

## 1. 日常の起動手順

### 1.1 基本起動（単一リポ）

```bash
cd ~/projects/ai-driven-dev-patterns
claude
```

### 1.2 マルチリポ起動（起動スクリプト使用）

```bash
# メインリポのみ
~/projects/start-workspace.sh

# 全リポを束ねて起動
~/projects/start-workspace.sh all

# 特定リポを追加
~/projects/start-workspace.sh dev-process-improvement
```

### 1.3 手動マルチリポ起動

```bash
cd ~/projects/ai-driven-dev-patterns
export CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD=1
claude --add-dir ../dev-process-improvement
```

> **注意**: `CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD=1` を設定しないと、追加リポの CLAUDE.md が読み込まれない。

## 2. セッション管理

### 2.1 セッション開始

```bash
# 新しいセッションを開始
claude

# 前回のセッションを再開（直近）
claude --continue

# 特定のセッションを再開
claude --resume
```

### 2.2 セッション中の操作

| 操作 | コマンド |
|------|---------|
| ヘルプ表示 | `/help` |
| セッション情報 | `/status` |
| コンテキストクリア | `/clear` |
| スキル一覧 | `/` + Tab |

### 2.3 セッション終了

- `Ctrl+C` または `/exit` で終了
- セッション履歴は自動保存される
- 次回 `claude --continue` で再開可能

## 3. Git ワークフロー

### 3.1 ブランチ運用

```bash
# 作業ブランチの作成
git checkout -b feature/my-feature

# 変更のコミット
git add <files>
git commit -m "変更内容の説明"

# リモートへプッシュ
git push -u origin feature/my-feature
```

### 3.2 PR 作成

```bash
# CLI から PR 作成
gh pr create --title "PRタイトル" --body "PR本文"

# PR 一覧確認
gh pr list

# PR の詳細確認
gh pr view <PR番号>
```

### 3.3 upstream 同期

upstream の最新変更を取り込む場合は `scripts/sync.sh` を使用する。
詳細は `.claude/rules/sync.md` を参照。

## 4. CodeSail 連携（オプション）

### 4.1 初回ペアリング

```bash
cd ~/projects/ai-driven-dev-patterns
codesail pair
# QR コードが表示される → iPhone の CodeSail アプリで読み取り
```

### 4.2 日常の使い方

1. WSL で Claude Code セッションを起動する
2. iPhone の CodeSail アプリでセッションを開く
3. チャットで指示を送信する

> セッションは WSL 側で起動しておく必要がある。iPhone からはセッション起動不可。

## 5. トラブルシューティング

### 5.1 Claude Code CLI の認証エラー

**症状**: `claude` 起動時に認証エラーが出る

```bash
# 認証状態を確認
claude auth status

# 再認証
claude auth login
```

API キーを使用している場合:
```bash
# 環境変数で設定
export ANTHROPIC_API_KEY="your-api-key"

# 永続化する場合は ~/.bashrc に追記
echo 'export ANTHROPIC_API_KEY="your-api-key"' >> ~/.bashrc
```

### 5.2 gh CLI の認証エラー

**症状**: `gh pr create` が 401 エラーになる

```bash
# 認証状態を確認
gh auth status

# 再認証
gh auth login
```

### 5.3 --add-dir で追加リポの CLAUDE.md が読み込まれない

**原因**: 環境変数が未設定

```bash
# 環境変数を設定して起動
export CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD=1
claude --add-dir ../other-repo

# 永続化する場合は ~/.bashrc に追記
echo 'export CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD=1' >> ~/.bashrc
```

### 5.4 CodeSail 接続不良

**症状**: iPhone から WSL のセッションに接続できない

1. WSL 側で Claude Code セッションが起動中か確認する
2. CodeSail CLI のバージョンを確認: `codesail --version`
3. ペアリングをやり直す: `codesail pair`
4. ネットワーク接続を確認（iPhone と PC が同一ネットワーク上にあること）

### 5.5 WSL が起動しない

```bash
# Windows PowerShell から
wsl --status
wsl --update
wsl --shutdown
# 再度 WSL を起動
```

### 5.6 git push が拒否される

**症状**: `remote: Permission to ... denied`

```bash
# リモート URL を確認
git remote -v

# HTTPS の場合は gh の認証を確認
gh auth status

# SSH の場合は鍵を確認
ssh -T git@github.com
```
