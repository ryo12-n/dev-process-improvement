# 実施計画: 個人PCローカル環境構築

## フェーズ構成

| フェーズ | 内容 | 担当 | 成功基準 |
|---------|------|------|----------|
| 1 | 基盤環境構築（WSL + Claude Code CLI + gh CLI） | 人間 | `claude --version` と `gh auth status` が成功する |
| 2 | 設定・構成ファイル準備（CLAUDE.md、起動スクリプト、settings.json） | L2-worker | 設定ファイル・スクリプトが作成され、レビュー済み |
| 3 | CodeSail 導入・運用ガイド整備 | 人間 + L2-worker | CodeSail ペアリング完了、運用ガイドが作成済み |

## スケジュール

1. **フェーズ1**: 人間が WSL + CLI をインストール（人間タスク。L2-worker の事前作業なし）
2. **フェーズ2**: L2-worker がマルチリポ用設定ファイル・起動スクリプトを作成（フェーズ1 完了後に実行）
3. **フェーズ3**: 人間が CodeSail を導入 + L2-worker が運用ガイドを作成（フェーズ2 と並行可能）

> **注意**: フェーズ1 は人間の手作業のため、L2-worker はフェーズ2 以降のタスクのみ担当する。ただし、フェーズ2 のタスク（設定ファイル設計等）はフェーズ1 の完了を待たずに着手可能。

## 成功基準（全体）

1. WSL 上で `claude --add-dir ../other-repo` によるマルチリポセッションが起動できる
2. WSL 上で `gh pr create` が成功する（認証済み状態）
3. iPhone の CodeSail アプリからセッションにチャット指示を送信できる
4. セッション起動スクリプト（`start-workspace.sh`）でワンコマンド起動ができる
5. 運用ガイド・トラブルシューティングガイドが作成されている

## リソース・前提条件

- Windows PC に WSL（Ubuntu）が導入済み、または導入可能な状態であること
- Anthropic API のアカウント・API キーが利用可能であること
- GitHub アカウントの認証情報（SSH キーまたはトークン）が利用可能であること
- `refs/個人PCローカル環境構築-手順書.md` を参照資料として使用する

## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| WSL 未導入で追加セットアップが必要 | 低 | 中 | 手順書に WSL 導入手順も含める。公式ドキュメント参照リンクを記載 |
| Claude Code CLI のインストール方法が変更されている | 低 | 中 | 公式ドキュメント（code.claude.com/docs/ja/setup）を最新確認する |
| CodeSail がサービス停止・仕様変更 | 低 | 低 | CodeSail は補助機能。CLI 直接操作で代替可能 |
| マルチリポ構成で CLAUDE.md の読み込み順に予期せぬ問題 | 中 | 中 | `--add-dir` 時の CLAUDE.md 読み込み動作を事前に調査し、設計に反映する |
