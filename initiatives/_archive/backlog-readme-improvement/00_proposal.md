# 改善施策提案: backlogのREADME改善

## 背景・課題

`backlog/README.md` の施策一覧テーブルが肥大化し、以下の問題が発生している。

1. **テーブルの崩れ**: 日本語ファイル名やリンクが長く、Markdown テーブルの可読性が低い
2. **プログラム的操作の困難さ**: Markdown テーブルではソート・フィルタ等の操作ができない
3. **情報の混在**: README に一覧テーブル・すみわけ説明・運用ルールが混在し、肥大化している

## 目標

- backlog 一覧を CSV 形式（`backlog/backlog.csv`）で管理し、可読性とプログラム的操作性を向上させる
- inbox・課題管理CSV・backlog のすみわけ説明を `docs/` に切り出し、README をエントリーポイントに絞る
- 運用ルール（トリアージワーカー等）の連動更新を完了する

## スコープ

### やること

- `backlog/backlog.csv` の新規作成（現 README テーブルからの移行）
- `docs/` へのすみわけガイド切り出し
- `backlog/README.md` のスリム化（エントリーポイント化）
- 運用ルール（`backlog/README.md` 内）の CSV 対応更新
- トリアージ関連スキル定義で backlog README テーブル更新を前提としている箇所の連動更新（該当があれば）

### やらないこと

- backlog ファイル（個別 `.md`）のフォーマット変更（別施策「backlogテンプレート凡例強化」のスコープ）
- 課題管理 CSV のフォーマット変更
- トリアージプロセス全体の見直し

## 期待される効果

- backlog 一覧の可読性向上（テーブル崩れの解消）
- ソート・フィルタ等のプログラム的操作が可能に
- README の行数削減（150行以内目標、ドキュメント階層方針に準拠）

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| CSV 更新の運用漏れ（README テーブルと同じ問題の再発） | 中 | 運用ルールを明確に記載し、トリアージで棚卸し対象にする |
| すみわけガイド切り出しで情報の発見性が低下 | 低 | README からのリンクを明示する |

## 壁打ちの背景

- CSV 配置先: `backlog/backlog.csv`（backlog ディレクトリ内に閉じる方針。ルート直下案もあったが、backlog の管理資料として閉じるのが自然）
- すみわけ説明: inbox・課題管理CSV・backlog の関係を説明する資料を `docs/` に切り出す（README からは削除し、リンクのみ残す）
- 連動更新: トリアージワーカー等の運用ルール変更も本施策スコープに含める
