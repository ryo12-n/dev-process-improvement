# 管理システムのすみわけガイド

本リポジトリでは **inbox**、**backlog**、**課題管理 CSV** という3つの管理システムを運用している。これらは異なる目的で使い分けられており、重複ではない。

---

## 1. inbox（`inbox/*.md`）

- **目的**: 人間起票のアイデア・提案の受付窓口
- **単位**: 1件 = 1ファイル（`.md`）
- **内容**: アイデア・気づき・提案・参照物ポインター（`ref_*.md`）
- **ライフサイクル**: 人間が起票 → トリアージで判断（施策化 / ルール反映 / 却下 / 保留）→ 処理済みは削除（git 履歴が証跡）
- **管理者**: 人間（起票）、トリアージセッション（処理・削除）

### 参照物（refs/）との関係

- 他PJの実装など実体ファイルを持ち込む場合は `refs/<名前>/` に本体を置き、`inbox/ref_<名前>.md` にポインターメモを置く
- ポインター（`inbox/ref_*.md`）の処理後も `refs/` の本体は削除しない

---

## 2. backlog（`backlog/entries/*.md` + `backlog/backlog.csv`）

- **目的**: トリアージで承認された施策候補の管理
- **単位**: 1施策 = 1ファイル（`entries/*.md`）。一覧は `backlog.csv` で管理
- **内容**: 施策名・優先度・ステータス・課題背景・期待効果・補足情報
- **ライフサイクル**: inbox からトリアージで承認 → backlog に追加 → initiative 開始時に `git rm` で削除（内容は `00_proposal.md` に引き継がれる）
- **管理者**: トリアージセッション（追加）、L1マネージャー（施策開始時の削除）

---

## 3. 課題管理 CSV（`プロセス改善_課題管理.csv` + `issues/entries/ISS-XXX.md`）

- **目的**: 施策横断の課題トラッキング
- **単位**: 1課題 = CSV の1行（`ISS-XXX` 番号で管理）+ 詳細ファイル
- **内容**: ID・起票日・カテゴリ・種別・タイトル・課題詳細・対応方針案・ステータス・優先度・関連施策・詳細ファイル
- **ライフサイクル**: 施策実施中に発見 → `07_issues.md` に一時記録 → 施策横断の課題は CSV に転記（`issues/entries/ISS-XXX.md` を作成）→ 対応完了でクローズ
- **管理者**: L2ワーカー（起票・転記）、L1マネージャー（ステータス更新）、トリアージセッション（棚卸し）

---

## 4. 使い分けの基準

| 観点 | inbox | backlog | 課題管理 CSV |
|------|-------|---------|-------------|
| 起票者 | 人間 | トリアージセッション | L2ワーカー / L2評価者 |
| 粒度 | アイデア・気づき | 施策候補（やるべきこと） | 課題（気づき・問題・改善提案） |
| スコープ | 未分類 | 単一施策 | 施策横断 |
| 採番 | ファイル名 | ファイル名 + CSV行 | ISS-XXX |
| 状態管理 | 未処理 / 処理済（削除） | ステータス欄（候補 / initiative 開始済） | ステータス列（起票 / 方針確定 / タスク化済 / クローズ / 対応不要） |
| 判断者 | トリアージセッション | L1マネージャー | L1マネージャー |

### フロー概要

```
人間のアイデア → inbox/ → トリアージで判断
                              ├→ 施策化 → backlog/ → initiative 開始 → sessions/initiatives/
                              ├→ ルール反映 → .claude/rules/ or skills/
                              └→ 却下・保留

AI発見の課題 → 07_issues.md → 施策完了時に判断
                              ├→ 施策横断 → CSV + issues/entries/ISS-XXX.md
                              └→ 施策内完結 → 転記不要
```
