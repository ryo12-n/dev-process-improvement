## 1. プロジェクト概要

社内の蔵書を管理するための REST API サーバーを構築する。書籍の登録・検索・貸出・返却を API 経由で行い、フロントエンドや Slack Bot など複数のクライアントから利用できるようにする。

## 2. 背景・動機

現在、社内の蔵書は Google スプレッドシートで管理されており、以下の課題がある。

- 貸出・返却の記録が手作業のため記入漏れが多い
- 同時編集で競合が発生し、データが壊れることがある
- 検索が遅く、蔵書数が 5,000 冊を超えたあたりから実用に耐えない
- 外部ツール（Slack Bot 等）から自動的にデータを参照・更新できない

API 化することで、データの整合性を担保しつつ、複数クライアントからの統一的なアクセスを実現する。

## 3. ユーザーストーリー

- [x] 社員として、ISBN を入力して書籍を登録したい。なぜなら手入力のミスを減らし、書誌情報を自動取得したいからだ。
- [x] 社員として、タイトル・著者・ISBN で蔵書を検索したい。なぜなら読みたい本が社内にあるかすぐに確認したいからだ。
- [x] 社員として、書籍を貸出予約したい。なぜなら他の人が借りている本が返却されたときにすぐ借りたいからだ。
- [x] 社員として、自分の貸出履歴を確認したい。なぜなら返却期限を把握して延滞を防ぎたいからだ。
- [x] 管理者として、貸出状況の統計レポートを取得したい。なぜなら蔵書の購入計画に活用したいからだ。
- [x] 管理者として、延滞者に自動でリマインドメールを送りたい。なぜなら手動連絡の手間を省きたいからだ。

## 4. 機能要件（概要レベル）

- 書籍の CRUD 操作（登録・取得・更新・削除）
- ISBN による書誌情報の自動取得（OpenBD API 連携）
- 蔵書検索（タイトル、著者、ISBN、タグによるフィルタリング・ページネーション）
- 貸出・返却の管理（貸出登録、返却登録、貸出状況の確認）
- 貸出予約機能（予約登録、予約キャンセル、予約順の管理）
- ユーザー管理（社員情報との連携、ロール管理: 一般/管理者）
- 貸出履歴の参照（個人別・書籍別）
- 延滞アラート（返却期限超過時のメール通知）
- 統計 API（月次貸出数、人気書籍ランキング、延滞率）

## 5. 非機能要件

| カテゴリ | 要件 |
|---------|------|
| パフォーマンス | API レスポンス 95 パーセンタイルで 200ms 以内。検索 API は 500ms 以内 |
| セキュリティ | JWT ベースの認証必須。HTTPS のみ。管理者 API は RBAC で保護 |
| 可用性 | 99.5%（営業時間帯: 平日 9:00-21:00）。計画メンテナンスは営業時間外 |
| スケーラビリティ | 同時接続 100 ユーザー。蔵書 50,000 冊まで対応 |

## 6. 技術制約・前提条件

- 言語/フレームワーク: TypeScript / Hono（Cloudflare Workers 対応）
- データベース: PostgreSQL（Supabase）
- インフラ: Cloudflare Workers + Supabase
- その他: OpenAPI 3.1 準拠の API ドキュメント自動生成。CI は GitHub Actions

## 7. 対象外（スコープ外）

- フロントエンド UI の実装（API のみ提供）
- Slack Bot の実装（API を利用する側は別プロジェクト）
- 書籍の物理的な棚管理（バーコードリーダー連携等）
- 課金・決済機能
- 多言語対応（日本語のみ）

## 8. 成功基準

- 全 API エンドポイントが OpenAPI 仕様に準拠し、自動テストで 90% 以上のカバレッジ
- ISBN 入力で書籍情報が自動取得され、登録までの手順が 3 ステップ以内
- 貸出・返却操作が API 1 コールで完了する
- 蔵書 10,000 冊のデータセットで検索 API が 500ms 以内にレスポンスを返す
- 延滞アラートメールが返却期限の翌日に自動送信される
