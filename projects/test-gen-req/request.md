## 1. プロジェクト概要

REST API で書籍管理システム（BookShelf API）を構築する。個人や小規模チームが蔵書を登録・検索・貸出管理できる Web API を提供する。

## 2. 背景・動機

チーム内の技術書・参考書が増えてきたが、誰がどの本を持っているか把握できず、重複購入や紛失が発生している。Slack で聞いて回る運用では限界があるため、蔵書を一元管理し貸出状況を可視化するシステムが必要になった。

## 3. ユーザーストーリー

- [ ] 管理者として、新しい書籍を ISBN で登録したい。なぜなら手入力を減らし正確な書誌情報を取り込みたいからだ。
- [ ] 利用者として、タイトル・著者・タグでキーワード検索したい。なぜなら目的の本を素早く見つけたいからだ。
- [ ] 利用者として、書籍の貸出リクエストを送信したい。なぜなら口頭やりとりをなくし記録を残したいからだ。
- [ ] 利用者として、自分の貸出中の書籍一覧を確認したい。なぜなら返却期限を把握したいからだ。
- [ ] 管理者として、貸出リクエストを承認・却下したい。なぜなら在庫状況を見て判断したいからだ。
- [ ] 管理者として、延滞中の貸出一覧を確認したい。なぜなら督促を効率化したいからだ。

## 4. 機能要件（概要レベル）

- 書籍の CRUD（登録・参照・更新・削除）
- ISBN による外部 API（OpenBD 等）からの書誌情報自動取得
- 書籍のタグ付け・カテゴリ分類
- キーワード検索（タイトル、著者、ISBN、タグ）
- 貸出リクエスト・承認・返却のワークフロー
- 貸出履歴の記録・参照
- ユーザー管理（管理者・一般利用者のロール）
- 延滞アラート（返却期限超過の一覧取得）

## 5. 非機能要件

| カテゴリ | 要件 |
|---------|------|
| パフォーマンス | 検索レスポンス 500ms 以内（1000冊規模） |
| セキュリティ | JWT ベースの認証・認可。パスワードは bcrypt でハッシュ化 |
| 可用性 | 99.5%（社内ツールのため高可用性は不要） |
| スケーラビリティ | 同時 50 ユーザー、蔵書 10,000 冊まで対応 |

## 6. 技術制約・前提条件

- 言語/フレームワーク: TypeScript + Express.js（チーム内の技術スタックに合わせる）
- データベース: PostgreSQL 15+
- インフラ: Docker Compose でローカル開発、将来的に AWS ECS へのデプロイを想定
- その他: OpenAPI 3.0 仕様でドキュメント化。CI は GitHub Actions を使用

## 7. 対象外（スコープ外）

- フロントエンド UI（API のみ提供。UI は別プロジェクトで対応）
- 電子書籍ファイルの保管・配信
- 外部図書館システムとの連携
- 多言語対応（日本語のみ）
- モバイルアプリ

## 8. 成功基準

- 全 API エンドポイントが OpenAPI 仕様通りに動作し、自動テストが通る
- ISBN 登録で書誌情報が自動取得できる
- 貸出ワークフロー（リクエスト→承認→返却）が一連の API 呼び出しで完結する
- 1000 冊規模の検索が 500ms 以内に完了する
