# 課題メモ（ワーカーセット別バッファ）: Set-2

> **このファイルの役割**: ワーカーセット内で発見された課題・知見・リスクの一時メモ置き場。
> マネージャーが `03_report.md` 作成時に全ワーカーセットの課題を集約し、`プロセス改善_課題管理.csv` へ転記する。

## 転記ルール

- ワーカー・評価者は課題をこのファイルに起票する（CSV 直接起票はしない）
- マネージャーが集約時に CSV 転記の要否を判断する
- 施策スコープ外・他のセッションでも再発しうる課題 → **マネージャーが CSV に転記**
- 当セッション内だけで完結する軽微な修正メモ → 転記不要（クローズ扱い）
- 転記済みの項目には `[転記済 ISS-XXX]` を末尾に追記する

## 未転記メモ

### [2026-03-11] gate_review ルーティング宣言の未実行率が25%
- **種別**: 改善提案
- **発見者**: ワーカー
- **内容**: MC-002/MC-005 の走査で、gate_review の横展開セクションに記載されたルーティング宣言55件（アクショナブル）のうち14件（25%）が実際には実行されていないことを検出。特に inbox ルーティングが11件未実行（未実行率29%）。gate_review のチェックリスト「ルーティング実行済みであることを確認した」にチェックが入っているにもかかわらず未実行のケースがある。
- **対応方針（案）**: l1-manager のクローズ手順において、ルーティング実行確認を「自己申告チェック」から「実在確認（ファイル・ISS-IDの存在を確認）」に強化する。具体的には、クローズ手順に「各ルーティング先にファイル/エントリが存在することを確認」ステップを追加する。
- **関連タスク**: MC-002, MC-005

### [2026-03-11] CSV への知見型ルーティングが転記漏れしやすい構造
- **種別**: 知見
- **発見者**: ワーカー
- **内容**: CSV ルーティング12件のうち、ISS-ID を明示しているもの（ISS-028, ISS-029, ISS-042 等）は100%転記済みだが、「種別: 知見」として ISS-ID なしで宣言された3件は全て未転記。知見型のルーティングは ISS-ID がないため転記の対象として認識されにくく、漏れやすい構造がある。
- **対応方針（案）**: gate_review の横展開セクションで CSV ルーティングを宣言する際、知見型であっても事前に ISS-ID を割り当てることを推奨する。または、クローズ手順で CSV ルーティング宣言と CSV 実エントリの突合チェックを行う。
- **関連タスク**: MC-002, MC-005

### [2026-03-11] MC-005 の inbox ルーティング実行確認でファイル名マッチの精度不足
- **種別**: 知見
- **発見者**: 評価者
- **内容**: MC-005 の未実行判定において、backlog-maintenance-session #1 が inbox 未実行と判定されたが、実際には `inbox/session-flow-checklist-precision.md` が現存し、ファイル内で発見元として当該 gate_review の必須把握事項 #1 を明記していた。同様に #2 も `inbox/triage-standard-policy-table-width.md` として現存。gate_review の知見タイトルと inbox ファイル名の語彙が異なる場合にキーワードマッチが失敗するパターン。
- **対応方針（案）**: メタ認知ワーカーの inbox 照合手順に「ファイル名マッチだけでなく、inbox 内ファイルの出典・発見元セクションとの内容ベース照合」を追加する。また、未実行一覧の14件→12件以下への修正が必要。
- **関連タスク**: MC-005
