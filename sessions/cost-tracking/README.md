# セッションコスト計測: キャンペーン運用ガイド

## 目的

施策のセッションコスト（トークン消費量・所要時間・リワーク率等）をサンプリング方式で計測し、プロセス改善の投資対効果をデータドリブンに評価する。

## ディレクトリ構成

```
sessions/cost-tracking/
├── README.md              ← 本ファイル（運用ガイド）
└── campaigns/
    └── YYYYMMDD.md        ← キャンペーンごとの横断比較レポート
```

## 計測テンプレート

`.claude/templates/session-cost-measurement.md` を参照。施策内の記録には Section B の `09_cost_record.md` 雛形を使用する。

## 計測サイクル

### トリガー条件

| トリガー | 条件 | アクション |
|---------|------|----------|
| N件完了トリガー | 施策が 3〜5件完了するごと | 標準キャンペーンを実施 |
| マイルストーントリガー | 重要なプロセス変更後（新セッション種別追加・テンプレート大幅改修等） | ターゲットキャンペーンを実施 |
| アドホック | コスト異常の疑いがある場合 | ミニキャンペーンを実施（特定施策のみ） |

N件の初期値は **3〜5件**。パイロットキャンペーン後に調整する。

### サンプル選定基準

1キャンペーンあたり **2〜3施策** をサンプリングする。

- 最低1件は最近完了した施策（フルライフサイクルデータの取得）
- 複雑度にバラつきを持たせる（S/M から1件、M/L から1件）
- 可能な限りセッション種別の多様性を確保する

## キャンペーン実施フロー

### 前方計測の場合（推奨）

1. L1 がサンプル対象の施策を選定する
2. 対象施策に `09_cost_record.md` を作成する（テンプレート Section B をコピー）
3. L2 に計測対象であることを伝達し、work_log にコストデータ行の追記を依頼する
4. 施策完了後、L1 が `09_cost_record.md` を集計する（Section C を記入）
5. `campaigns/YYYYMMDD.md` に横断比較レポートを作成する

### 事後計測の場合（フォールバック）

1. L1 が計測対象の完了済み施策を選定する
2. 各施策の work_log・コミット履歴・ゲートレビュー結果からデータを抽出する
3. 対象施策に `09_cost_record.md` を作成し、抽出データを記入する
4. `campaigns/YYYYMMDD.md` に横断比較レポートを作成する
