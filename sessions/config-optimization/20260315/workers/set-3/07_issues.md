### ISS-TEMP-006: Phase 2 の GHA スキル数カウント誤り（5件 → 6件）
- **発見フェーズ**: Phase 3
- **発見者**: config-proposal-worker
- **重要度**: 低
- **内容**: Phase 2 レポート（ISS-TEMP-005）では GHA スキルを5件（gha-close, gha-execute, gha-wallbash, gha-question, gha-test）と報告しているが、`gha-guideline` も `user-invocable: false` の GHA 関連スキルであり、合計6件が正しい。gha-guideline は GHA ワークフローの権限設計リファレンスであり、CLI セッションでの自動起動は不要
- **影響**: OPT-003 の対象スキル数が5件ではなく6件になる。影響は軽微（1ファイル追加）
- **推奨対応**: OPT-003 の提案に6件で反映済み。ISS-TEMP-005 の記載を6件に修正する必要あり
- **CSV転記**: [未転記]

### ISS-TEMP-007: @import のインライン展開によるコンテキスト削減の限界
- **発見フェーズ**: Phase 3
- **発見者**: config-proposal-worker
- **重要度**: 中
- **内容**: CLAUDE.md の @import 構文はインポート先ファイルを起動時にインラインで展開するため、コンテキスト消費量は分離前と変わらない。OPT-005 の CLAUDE.md 縮小提案において、@import で分離する23行（セッション終了時の気づき記録）+15行（ディレクトリ使い分けガイド）はコンテキスト削減に寄与しない。実際のコンテキスト削減効果があるのは rules/ への委譲（path-scoped による遅延読み込み）の42行分のみ
- **影響**: OPT-005 の期待効果の正確な理解に影響。「192行 → 112行に縮小」は CLAUDE.md ファイルの行数としては正しいが、コンテキスト消費量の削減は42行分のみ。残り38行分は可読性・保守性の改善効果のみ
- **推奨対応**: OPT-005 のレポートに明記済み。バックログエントリ作成時に @import の限界を注意事項として記載すること
- **CSV転記**: [未転記]

### ISS-TEMP-008: OPT-001 の Bash ツールチェックにおけるバイパスリスク
- **発見フェーズ**: Phase 3（評価時）
- **発見者**: config-optimizer-evaluator
- **重要度**: 中
- **内容**: OPT-001 の PreToolUse hook は Bash ツールのコマンド文字列を grep でパターンマッチングしているが、変数展開（`cat "$SECRET_FILE"`）、パイプ経由（`base64 .env | ...`）、エイリアス、サブシェル等の間接参照パターンを検出できない。単純な文字列マッチングでは Bash 経由のセンシティブファイルアクセスを完全にはブロックできない
- **影響**: OPT-001 単体ではセキュリティガードが不完全。Bash ツール経由のバイパスが理論上可能
- **推奨対応**: OPT-001 のリスク欄に段階的導入と sandbox（OPT-007）との二重防御が既に記載されている。実装タスク作成時に「Bash ツールの間接参照バイパスリスク」を注意事項として明記し、sandbox との併用を必須とすること
- **CSV転記**: [未転記]
