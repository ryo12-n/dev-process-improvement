# 課題メモ（ワーカーセット別バッファ）: Set-2

> **このファイルの役割**: ワーカーセット内で発見された課題・知見・リスクの一時メモ置き場。
> マネージャーが `03_report.md` 作成時に全ワーカーセットの課題を集約し、`プロセス改善_課題管理.csv` へ転記する。

## 転記ルール

- ワーカー・評価者は課題をこのファイルに起票する（CSV 直接起票はしない）
- マネージャーが集約時に CSV 転記の要否を判断する
- 施策スコープ外・他のセッションでも再発しうる課題 → **マネージャーが CSV に転記**
- 当セッション内だけで完結する軽微な修正メモ → 転記不要（クローズ扱い）
- 転記済みの項目には `[転記済 ISS-XXX]` を末尾に追記する

### [2026-03-12] backlog.csv とエントリファイルの優先度不一致（permission-prompt-optimization）
- **種別**: 知見
- **発見者**: ワーカー
- **内容**: `permission-prompt-optimization` エントリで、backlog.csv の優先度列は「低」だが、エントリファイル（`backlog/entries/permission-prompt-optimization.md`）のメタ情報テーブルでは「🟡 中」と記載されている。どちらが正しい優先度かが不明確。
- **対応方針（案）**: backlog.csv とエントリファイルの優先度を統一する。BM-004 の走査手順に「CSV とエントリファイルの優先度照合」ステップを追加し、今後の不一致を検出できるようにする。
- **関連タスク**: BM-004

### [2026-03-12] BM-004 優先度分布カウントの不一致（評価者検出）
- **種別**: 知見
- **発見者**: 評価者
- **内容**: BM-004 スキャンレポートの優先度分布サマリ（高2/中15/低35）が backlog.csv の実際の分布（高2/中16/低34）と一致しない。中が1件少なく、低が1件多い。変更提案反映後の数値も同様に不正確（レポート: 高5/中12/低35 → 正: 高5/中13/低34）。
- **対応方針（案）**: マネージャーが集約レポート作成時に正しい数値を使用する。ワーカーの走査手順に「定量サマリは backlog.csv から機械的に集計すること」を追加し、手動集計による誤差を防止する。
- **関連タスク**: BM-004
