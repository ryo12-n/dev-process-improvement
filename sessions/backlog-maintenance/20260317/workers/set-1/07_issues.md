# 課題メモ（ワーカーセット別バッファ）: Set-1

> **このファイルの役割**: ワーカーセット内で発見された課題・知見・リスクの一時メモ置き場。
> マネージャーが `03_report.md` 作成時に全ワーカーセットの課題を集約し、`プロセス改善_課題管理.csv` へ転記する。

## 転記ルール

- ワーカー・評価者は課題をこのファイルに起票する（CSV 直接起票はしない）
- マネージャーが集約時に CSV 転記の要否を判断する
- 施策スコープ外・他のセッションでも再発しうる課題 → **マネージャーが CSV に転記**
- 当セッション内だけで完結する軽微な修正メモ → 転記不要（クローズ扱い）
- 転記済みの項目には `[転記済 ISS-XXX]` を末尾に追記する

### [2026-03-17] CSV と .md エントリ間のステータス不一致（4件）
- **種別**: 不具合
- **発見者**: ワーカー
- **内容**: 以下の4件で backlog/entries/*.md のステータスが「完了」だが、backlog.csv のステータスが「候補」のまま不一致が発生している:
  - BL-004: ハーネスエンジニアリング観点でのブラッシュアップ（.md: 完了、CSV: 候補）
  - BL-010: backlogテンプレート凡例強化（.md: 完了、CSV: 候補）
  - BL-012: 知見集約テーブル発見元列のinitiatives逆移植（.md: 完了、CSV: 候補）
  - BL-072: 非施策セッション成果物の構造・プレフィックス見直し（.md: 完了、CSV: 候補）
  csv-conflict-prevention 施策で .md → CSV 自動生成（PreCommit hook）が実装されたはずだが、これら4件の .md ステータス変更時に CSV が再生成されていない。原因として以下が考えられる:
  1. PreCommit hook が未設定（hook のインストールが未実施）
  2. generate-csvs.py のステータスマッピング（STATUS_MAP）に「完了」のマッピングが欠落
  3. .md のステータス更新が hook 実装前に行われた（遡及的な再生成が未実施）
- **対応方針（案）**: `scripts/generate-csvs.py` を手動実行し CSV を再生成する。hook が正常に機能しているか検証する
