# 課題メモ（ワーカーセット別バッファ）: Set-3

> **このファイルの役割**: ワーカーセット内で発見された課題・知見・リスクの一時メモ置き場。
> マネージャーが `03_report.md` 作成時に全ワーカーセットの課題を集約し、`プロセス改善_課題管理.csv` へ転記する。

## 転記ルール

- ワーカー・評価者は課題をこのファイルに起票する（CSV 直接起票はしない）
- マネージャーが集約時に CSV 転記の要否を判断する
- 施策スコープ外・他のセッションでも再発しうる課題 → **マネージャーが CSV に転記**
- 当セッション内だけで完結する軽微な修正メモ → 転記不要（クローズ扱い）
- 転記済みの項目には `[転記済 ISS-XXX]` を末尾に追記する

### [2026-03-17] CSV 自動生成スクリプトのステータスマッピング不完全の疑い
- **種別**: 不具合
- **発見者**: ワーカー
- **内容**: csv-conflict-prevention 施策で CSV 自動生成（.md → CSV）が実装されたにもかかわらず、set-1 が BL-004/BL-010/BL-012/BL-072 の4件で CSV のステータスが「候補」のまま、.md では「完了」という不一致を検出した。CSV 自動生成が期待通りに機能していない可能性がある。set-1 でも同様の課題が起票されているが、BM-003 の前提変化チェック観点から csv-conflict-prevention の完了後も不一致が残存している事実は、同施策の成果物に品質課題がある可能性を示唆する。
- **対応方針（案）**: csv-conflict-prevention の自動生成スクリプトのステータス解析ロジックを確認し、.md のメタ情報テーブルからステータスを正しく抽出できているか検証する。特に「完了」ステータスが CSV に反映されないケースのパターンを特定する。
- **関連タスク**: BM-003（環境変化3: csv-conflict-prevention 完了）
- **関連**: set-1 07_issues.md の CSV/md ステータス不一致課題
