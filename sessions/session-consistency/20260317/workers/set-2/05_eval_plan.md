## 評価計画

### 評価対象ファイル

| # | ファイル | 評価観点 |
|---|---------|---------|
| 1 | `workers/set-2/04_scan_report.md` | 8ターゲットのカバレッジ、エビデンス品質、客観性、漏れ検出 |
| 2 | `workers/set-2/07_issues.md` | 課題起票の適切性、scan_report との整合性 |

### 評価基準

| # | 評価項目 | 基準 |
|---|---------|------|
| 1 | カバレッジ | 8つの SC ターゲット（SC-001〜SC-008）すべてが分析されているか |
| 2 | エビデンス品質 | 不整合が具体的なデータ（ファイル名、セクション名、行番号等）に基づいているか |
| 3 | 客観性 | 分析が事実に基づいており、推測が明示的にマークされているか |
| 4 | 漏れ検出 | 明らかな不整合がワーカーに見落とされていないか |
| 5 | リファレンスとの整合 | 分析が Phase 1 のリファレンスと整合しているか |

### 評価方法

1. scan_report の全8ターゲットセクションを読み、構造（Current State / Requirements / Gaps / Recommendations）の完備を確認
2. 不整合として報告された各項目について、実際のソースファイルを Grep/Read で検証
3. Phase 1 scan_report の検出事項が Phase 2 で適切に深掘りされているか照合
4. ワーカーが見落としている可能性のある不整合がないか、独自にスポットチェック
5. 07_issues.md の起票内容が scan_report の Gaps と整合しているか確認
