## 評価計画

### 評価対象ファイル

| # | ファイル | 評価観点 |
|---|---------|---------|
| 1 | `workers/set-3/04_scan_report.md` | Phase 3 改善提案の品質・網羅性・具体性 |
| 2 | `workers/set-2/04_scan_report.md` | Phase 2 不整合との対応関係の照合用（読み取りのみ） |

### 評価基準

| # | 評価項目 | 基準 |
|---|---------|------|
| 1 | アクション可能性 | 各提案がバックログエントリとして転記可能な具体性を持つか（Current State / Proposed Change / Expected Benefit / Backlog Entry Draft の記載） |
| 2 | 優先順位の妥当性 | Impact x Effort の評価が根拠付きか（Impact/Effort の判定理由が明記されているか） |
| 3 | リスク評価 | 高インパクト提案にリスクと緩和策が記載されているか |
| 4 | 網羅性 | Phase 2 で特定された全不整合（SC-003: 3件、SC-006: 2件、SC-007: 2件、SC-008: 3件）に対して提案があるか |
| 5 | Quick Wins の特定 | 高 Impact / 低 Effort の提案が適切に特定されているか |

### 評価手順

1. Phase 2 の不整合10件と Phase 3 の提案8件のマッピングを確認し、網羅性を検証する
2. 各提案の構造（Current State / Proposed Change / Expected Benefit / Backlog Entry Draft）を確認し、バックログエントリとしての転記可能性を評価する
3. Impact x Effort 評価の根拠を個別に検証する
4. 高 Impact 提案（IMP-001, IMP-007）のリスク・緩和策を確認する
5. Quick Wins の選定基準と妥当性を検証する
6. 横断的提案（IMP-005〜IMP-008）の追加価値を評価する
