# 改善施策提案: dev-workflow-detail L1参照精査

## 背景・課題

施策 `workflow-doc-consistency-fix` において `dev-workflow-detail.md` 内の「施策管理 L1」参照はユーザーに置換済みだが、エスカレーションフロー（§4.3）や分岐条件テーブル（§3）内に一般的な「L1」ロール名としての参照が複数残存している。

残存箇所（起票時点）:
- §4.3 mermaid図（466-467行）
- テンプレート記述（487行）
- Level 3条件（499行）
- 差し戻し上限（503行）
- §3 分岐条件テーブル（158, 237, 321行）

「L1」はセッション種別（マネージャーセッション）のロール名であり、設計方針上の最上位オーケストレーターの表記と一致しているか精査が必要。また 2026-03-12 の BM-003 検出により、`manager-common-policy` 整備によって L1/L2 の役割定義が共通ポリシーに移行しており、この変化を踏まえたスコープ再定義が前提条件となっている。

## 目標

`dev-workflow-detail.md` 内の残存「L1」参照が設計方針（`manager-common-policy` での L1 の位置づけ）と整合しているかを精査し、必要に応じて修正または現状維持の判断根拠を記録する。

## スコープ
### やること
- `dev-workflow-detail.md` の残存「L1」参照箇所（上記7箇所）を特定・確認
- `manager-common-policy` の L1 定義を確認し、整合性を判定
- 修正が必要な箇所は適切なロール名・表記に修正
- 修正不要と判断した箇所はその根拠を記録

### やらないこと
- `dev-workflow-detail.md` 以外のファイルへの変更（スコープ外）
- `manager-common-policy` 自体の変更
- 他の「L1」関連文書（`workflow.md` 等）の一括修正（別施策として検討）

## 期待される効果

## 壁打ちの背景

- BL-002 として起票された際は単純な「L1」→別表記への置換施策と見なせたが、2026-03-12 の BM-003 検出（manager-common-policy 整備）によりスコープが変化した
- `manager-common-policy` では L1 はマネージャーセッション種別の総称として使われており、`dev-workflow-detail.md` での用法が同義か、あるいは「施策管理 L1（= ユーザー）」という旧来の文脈で使われているかの判別が必要
- 精査の結果「現状のまま L1 表記で問題なし」という判断になる可能性もあるため、「修正」よりも「精査と判断記録」を主目的とするスコープで施策化した
