### 評価時の観点（T-001 の補足ガイダンス）

| Skill | 主な確認観点 |
|-------|------------|
| `/simplify` | Markdown 文書のリファクタ・冗長性削減に使えるか。3エージェント並列レビューがプロセス文書にも有効か |
| `/batch` | `docs/` や `.claude/skills/` 配下の一括更新に使えるか。例: 全 skills のメタデータフォーマット統一 |
| `/debug` | セッション失敗時のトラブルシューティングに使えるか。L2 サブエージェントの問題診断に有用か |
| `/loop` | 定期監視（PR 状態チェック、ブランチ状態確認等）に使えるか。具体的な活用シーンを特定 |
| `/claude-api` | API リファレンス参照が必要な場面があるか。dev-process-improvement では API 利用場面が限定的か |
