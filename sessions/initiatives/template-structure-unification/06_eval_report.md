## 発見された課題・改善提案

### 課題1: Wave 1 リストと T-002 の表記不整合

- **重要度**: 低（実害なし）
- **内容**: Wave 1 の箇条書きリスト（行244）に T-002 が含まれていないが、直後の本文（行248）では「T-001〜T-007 の7タスクは...全タスクを Wave 1 で並列実行可能」と記述。箇条書きは T-001, T-003, T-004, T-005, T-006, T-007 の6タスクのみ。
- **推奨対応**: L1 が `02_tasks.md` 作成時に T-002 の Wave 配置を明確化する。(a) 箇条書きに T-002 を追加、または (b) T-002 を Wave 1.5（T-001 完了後に開始）として別出し。

### 課題2: T-002 の Prerequisite と Wave 配置の論理的不整合

- **重要度**: 低（正当化の説明あり）
- **内容**: T-002 の Prerequisite tasks に「T-001」が設定されているが、Wave Assignment では T-001 と同じ Wave 1 に配置されている。計画ワーカーは「参照元情報は `01_plan.md` で把握可能」と正当化しているが、Prerequisite フィールドの設定と矛盾する。
- **推奨対応**: (a) Prerequisite を「None」に変更し Read-only dependencies のみで表現する、または (b) Wave を分離して T-002 を Wave 1.5 に配置する。どちらの対応でも実装への影響はない。

### 課題3: commit-message.md の除外判断が明示されていない

- **重要度**: 低
- **内容**: `00_proposal.md` のスコープに「commit-message.md のファイル参照があれば更新」と記載があるが、`01_plan.md` の変更対象8ファイルおよび `02a_task_division.md` のタスクには含まれていない。実際に `commit-message.md` には旧ファイル名（`03_work_log.md`、`04_work_report.md`）への参照がコミット例として存在する。
- **影響**: コミット例は illustrative（例示）であり prescriptive（規定）ではないため、実害は軽微。ただし除外判断の根拠が文書化されていない。
- **推奨対応**: (a) `01_plan.md` のスコープ外セクションに除外理由を追記、または (b) T-002 等の既存タスクに `commit-message.md` の例文更新を追加。
