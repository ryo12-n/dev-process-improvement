### [2026-03-13] EVL-002: Phase 2 の gh issue close/edit で ${{ }} 直接展開 [転記済 ISS-058]
- **種別**: 改善提案
- **内容**: `backlog-auto-execute.yml` の "Close issue" ステップで `${{ github.event.issue.number }}` をシェルコマンドに直接展開している。`github.event.issue.number` は整数値であるため実害リスクは極めて低いが、同ワークフローの Phase 1 で採用した env 経由パターンと一貫性がない。今後 GHA ワークフローを追加する際にこのパターンがコピーされると、文字列値の場合にインジェクションリスクになりうる。
- **対応方針（案）**: `ISSUE_NUMBER: ${{ github.event.issue.number }}` を `env:` に追加し、`run:` 内では `${ISSUE_NUMBER}` を使用する。GHA ワークフロー作成時のチェックリストに「${{ }} をシェルコマンドに直接展開しない」を追加する
- **関連タスク**: T-003
- **転記判断**: CSV に転記（GHA ワークフロー全体に適用すべきパターンであり、施策をまたいで再発しうる）
