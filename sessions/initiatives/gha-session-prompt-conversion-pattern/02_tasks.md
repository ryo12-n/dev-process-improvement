## T-002 詳細仕様

### 7.1 変換の全体フロー

CLI SKILL.md → GHA SKILL.md + workflow YAML への変換手順を5ステップで記載する:
1. CLI SKILL.md から GHA で自動化したいフェーズ/責務を特定する
2. GHA 専用 SKILL.md を `.claude/skills/gha-<name>/SKILL.md` に作成する（7.2 のテンプレートに従う）
3. ワークフロー YAML を `.github/workflows/<name>.yml` に作成する（7.3 のプロンプトテンプレートに従う）
4. 権限3層モデル（セクション1）の整合性を確認する
5. セクション5の権限マトリクスに追記する

### 7.2 GHA SKILL.md テンプレート

既存5スキルの共通構造をテンプレートとして示す。必須セクション:
- frontmatter（name, description, user-invocable: false）
- コンテキスト（正当性の宣言 — 定型文）
- あなたの役割（CLI SKILL.md から対象フェーズの責務のみを抽出・限定）
- 作業フロー（ステップバイステップ。動的コンテキストの参照方法を含む）
- やること（明示的な許可リスト）
- やらないこと（明示的な禁止リスト — 共通4項目: PR作成禁止、gh禁止、破壊的git禁止、force push禁止）
- 制約（対象リポジトリ、コミットメッセージ規約等）

参照実装: `gha-wallbash/SKILL.md`

### 7.3 ワークフロー YAML プロンプトテンプレート

最小プロンプトの定型を示す:
```yaml
prompt: |
  `.claude/skills/gha-<name>/SKILL.md` を Read ツールで読み取り、その指示に従って作業してください。
  このタスクは GitHub Actions の自動化ワークフローから実行されています。

  ## 動的コンテキスト
  [ワークフロー inputs / 前ステップ outputs を列挙]
```

動的コンテキスト注入の3パターン:
1. `${{ inputs.x }}` — ワークフロー inputs
2. `${{ steps.x.outputs.y }}` — 前ステップの出力
3. 一時ファイル（`/tmp/issue_body.txt`）— 長文コンテンツ

セクション4.4（`$(...)` はリテラル）への相互参照を含める。

### 7.4 後続 Shell ステップの設計

Claude 実行後の Shell ステップパターンをテーブルで示す:
- サマリー抽出（execution_file → jq）
- PR 作成（gh pr create）
- Issue コメント投稿（gh issue comment）
- ラベル操作（gh issue edit --add-label/--remove-label）

セクション2（Claude 委任 vs Shell ステップ）への相互参照を含める。
