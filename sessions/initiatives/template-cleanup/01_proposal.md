# 改善施策提案: template-cleanup

## 背景・課題

施策ディレクトリに不要なテンプレートファイルが残り続けている。直近5施策を調査した結果、以下2種類の不要ファイルを特定した。

1. **`phase-*/01_gate.md`**: 5施策中4施策でプレースホルダー（`[施策名]`・`YYYY-MM-DD`）が未記入のまま。ゲート判定は `05_gate_review.md` で行われており、事実上使われていない
2. **`phase-*/_template/`**: `set-N/` にコピーされた後も削除されず残存。全施策で確認（7ファイル × 2フェーズ = 14ファイル）

## 目標

- 施策ディレクトリから不要テンプレートファイルを構造的に排除する
- テンプレートコピー後の手作業を減らし、施策ディレクトリをクリーンに保つ

## スコープ
### やること
- `_template` から `phase-*/01_gate.md` を削除する
- L1 マネージャースキルの set コピー手順に `_template/` 削除ステップを追加する
- `parallel-dev.md` の set コピー方式に `_template/` 削除を明記する

### やらないこと
- 既存アーカイブ（125施策）の一括清掃
- テンプレート構造の大幅な再設計

|-------------|------------|-------------|
| なし | - | - |

## 期待される効果
- 施策あたり不要ファイル16個（01_gate.md × 2 + _template/ 内ファイル × 14）が排除される
- ゲート判定の情報源が `05_gate_review.md` に一元化される

## リスク
| リスク | 影響度 | 対策 |
|--------|--------|------|
| 01_gate.md を使っている施策がある可能性 | 低 | 調査済み: 直近5施策中、記入されていたのは1施策（template-composition-spec）のみ。05_gate_review.md と重複 |

## 壁打ちの背景
- `phase-*/01_gate.md` と `05_gate_review.md` の使い分けを調査した結果、01_gate.md はほぼ未使用であり廃止が妥当と判断
- `_template/` 削除のタイミングは「set-N コピー直後」が最もシンプル。テンプレートのルート `_template` から除外する方式と比較し、追加の set が必要になった場合にルートの `_template` から再コピーすれば対応可能
