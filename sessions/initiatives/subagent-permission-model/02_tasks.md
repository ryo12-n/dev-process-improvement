### タスク詳細

#### T-001: allow リスト継承の実機検証

L2-worker 自身がサブエージェントとして動作しているため、自身の環境で以下を実行し許可プロンプトの発生有無を観察する:

1. **allow リストに含まれるコマンド**: `ls`, `git status`, `cat`, `echo`, `cp` 等 → 許可プロンプトなしで実行されるか
2. **allow リストに含まれないコマンド**: `date`, `pwd`, `whoami` 等 → 許可プロンプトが発生するか
3. **deny リストに含まれるコマンド**: `rm -rf` 等 → 拒否されることを確認

各カテゴリの結果を表形式で `03_work_log.md` に記録すること。

#### T-002: tools フィールドと権限の関係検証

1. 現在のエージェント定義（`l2-worker.md`）の `tools: ["Read", "Write", "Edit", "Glob", "Grep", "Bash"]` で各ツールが利用可能か確認
2. `tools` フィールドに含まれていないツール（`WebSearch`, `WebFetch` 等）がサブエージェント内で利用可能か確認
3. `disallowedTools` の挙動はドキュメント調査結果（`00_proposal.md` の壁打ちの背景）を参照して記録

#### T-003: バックグラウンド実行時の権限制約の調査

1. 本PJの L2 サブエージェント起動方式を確認（L1 SKILL.md の記述を基に判断）
2. フォアグラウンド実行であることを確認し、T-001 の結果と照合
3. バックグラウンド実行（`run_in_background`）時のプリアプルーブ設定方法を公式ドキュメントから調査
4. 実機検証が困難な場合は `07_issues.md` に起票して完了とする

#### T-004: 運用ガイド作成

`docs/subagent-permission-guide.md` に以下の5項目を含めて作成:
1. サブエージェントの権限継承モデルの概要（Permission 評価順序）
2. `.claude/settings.json` の allow/deny リストとサブエージェントの関係
3. エージェント定義の `tools` / `disallowedTools` フィールドの役割
4. フォアグラウンド vs バックグラウンド実行時の挙動差
5. 本プロジェクトにおける推奨設定と注意点

参考情報: `00_proposal.md` の壁打ちの背景 + T-001〜T-003 の検証結果

#### T-005: 改善提案

`04_work_report.md` の「所感・次フェーズへの申し送り」セクションに以下をまとめる:
1. `.claude/settings.json` の allow リストに追加・削除が必要なパターン
2. エージェント定義に `permissionMode` や `disallowedTools` の明示記述を追加すべきか
3. deny リストの強化の必要性（サブエージェントからの意図しない操作防止）
4. 各提案に「変更すべき理由」「変更しない場合のリスク」を併記
