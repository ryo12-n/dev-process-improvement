# 実施計画: CLI環境での権限継承の再検証

## フェーズ構成

| フェーズ | 内容 | 成功基準 |
|---------|------|----------|
| 1 | CLI 環境での実機検証（deny/allow/tools） | 3項目 × 2環境の検証結果が記録されている |
| 2 | ドキュメント・設定更新 | ガイドと settings.json が検証結果と整合している |

## 成功基準（全体）
1. CLI 対話型・Task ツール経由の両環境で deny/allow/tools の動作が検証されている
2. `docs/subagent-permission-guide.md` の適用範囲テーブルが実機検証結果で裏付けされている
3. `.claude/settings.json` の設定が検証結果に基づき最適化されている
4. SDK/CLI 間の差異が正確に文書化されている

## リソース・前提条件
- 前施策のアーカイブ（`sessions/initiatives/_archive/subagent-permission-model/`）を参照可能
- ISS-045, ISS-046 の知見を前提とする
- Claude Code CLI 環境での検証が可能（本リポジトリの `.claude/settings.json` が適用される環境）

## リスクと対策
| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| deny 対象コマンドが実行されてしまう | 低 | 低 | 安全なコマンド（`echo "test"` 等）で代替検証 |
| Task ツール起動時に予期しない権限エラー | 中 | 低 | エラー内容を記録し知見として反映 |
