### 参考情報（文脈依存の気づき・今後の参考）

| # | 知見 | 背景・文脈 |
|---|------|-----------|
| 1 | hook スクリプトの `set -euo pipefail` と全パス exit 0 の設計が堅実 | `set -e` により途中エラーで即座に停止するが、各コマンドの失敗は `|| { ... }` で捕捉して output_result 経由で exit 0 を返す設計。hook がセッション開始をブロックしないという要件を満たしつつ、エラー情報を JSON で報告できる |
| 2 | rules ファイルの「設計仕様リファレンス」としての再利用パターン | ロジックを hook に移行した後も、external-repo-cleanup スキルが「元ロジック」として参照しているため rules ファイルを削除せず、バナーで役割を再定義した。ロジック移行時に参照元を確認する習慣は他施策でも有用 |
