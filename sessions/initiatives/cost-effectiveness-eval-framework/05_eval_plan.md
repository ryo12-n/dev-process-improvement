## 評価項目

| # | 評価項目 | 評価方法 | 判定基準（合格ライン） |
|---|---------|---------|---------------------|
| E-01 | タスク粒度 | 各タスクが1人のワーカーで完結可能なサイズか確認 | 全タスクが S〜M 複雑度で、単一ワーカーが1セッションで完了可能 |
| E-02 | CREATE/MODIFY/READ アノテーション | 全ファイルに対するアクセス種別がソース・提案書の記載と一致するか検証 | 全アノテーションが正確。抜け漏れなし |
| E-03 | コンフリクトマトリクス | Matrix 内の全セル（ファイル×タスク）を網羅的に検証し、アノテーションとの一致を確認 | Matrix が全ファイル・全タスクを網羅。競合検出の結論が正確 |
| E-04 | 依存関係 | Prerequisite tasks の設定が実際のデータフロー（成果物参照関係）と一致するか検証 | 依存関係が正確。循環依存なし。不足・過剰な依存なし |
| E-05 | Wave 割当 | 逐次ディスパッチの妥当性を、並列化可能性・オーバーヘッドの観点から評価 | Wave 割当が依存関係と整合。逐次ディスパッチの判断に根拠がある |
| E-06 | 完了条件 (Acceptance Criteria) | 各タスクの AC が SC-1〜SC-3 を充足するか、具体的・検証可能か確認 | 全 AC が具体的・検証可能。SC-1〜SC-3 へのトレーサビリティあり |
| E-07 | ドメインエージェント推奨 | Worker/Evaluator への推奨が施策内容に適しているか確認 | 推奨が妥当（または「なし」の根拠が明確） |
| E-08 | SC-1 充足度（計画レベル） | T-001 の AC が SC-1 の全要素（4軸定義・算出式・テーブルテンプレート・カスタマイズ指針）をカバーしているか | 全要素が AC に反映されている |
| E-09 | SC-2 充足度（計画レベル） | T-002 の AC が SC-2 の grep コマンド要件を正確に反映しているか | grep コマンドの期待文字列と AC の記載が一致 |
| E-10 | SC-3 充足度（計画レベル） | T-003 の AC が SC-3 の2つのテーブル（ディレクトリ構成・文書分類ポリシー）への追加を網羅しているか | 両テーブルへの追加が AC に含まれている |
| E-11 | メタルール横断検証タスク（T-004） | T-004 の AC が CLAUDE.md 変更の影響確認として十分か。既存スキル・ルールとの整合性検証が含まれているか | docs/workflow.md の更新要否判断、.claude/rules/ と .claude/skills/ との整合性確認が AC に含まれている |
| E-12 | メタルール横断検証チェックリスト | 施策が CLAUDE.md を変更するため、メタルール横断検証チェックリストの適用対象。チェック項目（フロー記述整合性・workflow.md 同期・TG-008 連動性）を評価 | チェックリストの3項目すべてを確認 |
