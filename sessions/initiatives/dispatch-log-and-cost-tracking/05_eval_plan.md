## 評価項目

| # | 評価項目 | 評価方法 | 判定基準（合格ライン） |
|---|---------|---------|---------------------|
| E-01 | タスク分割の網羅性（成功基準1〜5のスコープカバレッジ） | `01_plan.md` の成功基準5項目 × `02a_task_division.md` のタスク6件を突合し、各成功基準がいずれかのタスクの Acceptance criteria に反映されているか確認 | 成功基準5項目すべてが少なくとも1タスクの完了条件に対応していること |
| E-02 | CREATE/MODIFY/READ アノテーションの正確性 | 各タスクの Assigned files と Read-only dependencies を、`00_proposal.md` のスコープおよび実際のファイルシステム上の存在有無と照合 | 全ファイルアノテーションが正確（CREATE 対象が未存在、MODIFY 対象が既存、READ 対象が存在） |
| E-03 | Conflict Check Matrix の正確性 | マトリクスの各セルを Task Assignments のファイルアノテーションと1セルずつ照合。同一ファイルに複数 CREATE/MODIFY がないことを確認 | マトリクスの全セルが Task Assignments と一致し、競合検出結果が正しいこと |
| E-04 | 依存関係と Wave Assignment の妥当性 | Prerequisite tasks の設定を Conflict Check Matrix の READ 依存と照合し、循環依存がないことを確認。Wave 間の順序が依存関係に整合しているか検証 | 循環依存なし、Wave 内タスクが相互に依存していないこと、Wave 間の順序が Prerequisite tasks と整合 |
| E-05 | タスク粒度の適切さ | 各タスクが1ワーカーで完結可能か、Acceptance criteria が具体的・検証可能かを確認 | 全タスクが単独ワーカーで完結可能で、Acceptance criteria が曖昧でないこと |
| E-06 | メタルール横断検証チェックリスト | 本施策が `_template/` および `.claude/skills/` 配下を変更するため、メタルール横断検証3項目（フロー記述整合・workflow.md 同期・TG-008 基準連動）がタスクとしてカバーされているか確認 | 3項目すべてがタスクの Acceptance criteria に含まれていること |
| E-07 | 計画の成功基準に対する達成度（総合） | 成功基準5項目それぞれについて、タスク分割が実行された場合に達成可能かを評価 | 5項目中4項目以上が「達成見込み」であること |
