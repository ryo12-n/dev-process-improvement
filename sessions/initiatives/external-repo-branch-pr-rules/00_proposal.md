# 改善施策提案: 外部リポジトリ変更時のブランチ・PR運用ルール整備

## 背景・課題

ai-research-notebook-setup 施策において、外部リポジトリ（ai-research-notebook）への変更を main ブランチに直接コミットし、push も漏れるという問題が発生した。

現在のルール群（session-start-branch-cleanup, commit-message, l1-manager, l2-worker）は dev-process-improvement リポジトリ内の作業を前提としており、外部リポジトリへの変更手順が定義されていない。

## 目標

外部リポジトリへの変更もブランチ・コミット規約・PR ベースで管理されるようルールを整備し、main 直接コミットと push 漏れを防止する。

## スコープ
### やること
- `00_proposal.md` テンプレートに「外部リポジトリ」セクションを追加
- commit-message ルールの適用対象を外部リポジトリに拡張
- session-start-branch-cleanup に外部リポジトリの整理ステップを追加
- l1-manager スキルにフェーズ0・L2起動・成果物確認・クローズ手順の外部リポ対応を追加
- l2-worker エージェント定義に外部リポジトリでの作業手順を追加
- docs/workflow.md を更新

### やらないこと
- 外部リポジトリ固有のルール策定（各リポの個別事情には踏み込まない）
- sync-manager / fork-sync スキルの変更（同期フローは別施策の責務）

## 外部リポジトリ

| リポジトリ名 | ローカルパス | 変更内容の概要 |
|-------------|------------|-------------|
| なし | - | - |

## 期待される効果

- 外部リポジトリへの変更がブランチ・PR ベースで管理され、レビュー可能になる
- push 漏れが防止される
- 変更の追跡性が向上する

## リスク
| リスク | 影響度 | 対策 |
|--------|--------|------|
| SDK 環境で外部リポにアクセスできない | 中 | L2-worker に「パスが存在しない場合は 07_issues.md に起票して停止」ルールを追加 |
| アクティブ施策走査によるセッション開始時のオーバーヘッド | 低 | アクティブ施策は通常 0〜3 件のため影響は軽微 |

## 壁打ちの背景

- 「外部リポジトリ」は `--add-dir` で参照する任意のリポジトリを指す（特定リポに限定しない）
- commit-message 規約は外部リポにも同じものを適用する
- ブランチ命名も `initiative/<施策名>` を踏襲する
- session-start-branch-cleanup は外部リポにも適用する（アクティブ施策の `00_proposal.md` を参照して対象を特定）

## 備考・設計パターン

- `00_proposal.md` の「外部リポジトリ」セクションが全ての起点（ブランチ整理・L2起動・クローズ手順がこのセクションを参照する）
- L1 がフェーズ0 で外部リポのブランチを作成し、L2 はそのブランチにチェックアウトして作業する（ブランチ作成の責務は L1）
