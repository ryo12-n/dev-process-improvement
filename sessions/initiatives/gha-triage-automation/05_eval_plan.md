## 評価項目

| # | 評価項目 | 評価方法 | 判定基準（合格ライン） |
|---|---------|---------|---------------------|
| 1 | YAML 構文の正しさ | python3 yaml.safe_load で構文検証 | エラーなくパース可能 |
| 2 | schedule + workflow_dispatch トリガー | YAML 構造を検査 | 両トリガーが設定されている |
| 3 | 空振り対策（inbox 0件時の早期終了） | ワークフローステップの if 条件を検証 | 0件時に Claude Code 実行ステップがスキップされる |
| 4 | セキュリティ（secrets 管理・権限スコープ） | secrets 参照と permissions を検査 | ANTHROPIC_API_KEY が secrets 経由、permissions が最小権限 |
| 5 | ブランチ戦略と PR 作成 | ワークフローステップを検証 | triage/YYYYMMDD ブランチ作成と PR 作成ステップがある |
| 6 | プロンプトの SKILL.md 網羅性 | triage-manager SKILL.md との比較 | 作業フロー全体（事前調査〜ゲート判定）がカバーされている |
| 7 | 非対話モード不要ステップの除外 | プロンプト内容を検査 | ブランチ整理・壁打ち・ユーザー確認待ち・todo登録が除外されている |
| 8 | 単一エージェント逐次実行の設計妥当性 | プロンプトの代替設計を検証 | サブエージェント不使用でもトリアージフローが機能的に十分 |
| 9 | TG タスクの記述具体性 | 各 TG タスクの実行手順を検査 | TG-001〜TG-009 の手順が具体的で実行可能 |
| 10 | ステップ間の条件分岐（if 条件） | 各ステップの if 式を検証 | inbox 0件時とchanges なし時の分岐が正しい |
