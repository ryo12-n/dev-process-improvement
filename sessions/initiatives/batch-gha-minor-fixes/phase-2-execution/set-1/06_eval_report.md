## 発見された課題・改善提案

### 既存スキルとテンプレートの表現差異

評価項目#4の検証で、セクション7.7 の用途クラス別テンプレートと既存 GHA スキルの「やらないこと」セクション間に以下の差異を確認した。

| スキル | 用途クラス | 差異の内容 |
|--------|----------|-----------|
| gha-close | 書き込み系 | 2項目目が「`gh` コマンドによる Issue クローズ・ラベル操作」（テンプレートは「Issue・PR 操作」） |
| gha-execute | 書き込み系 | 2項目目が「`gh` コマンドによる Issue・PR・ラベル操作」（テンプレートは「Issue・PR 操作」） |
| gha-question | 読み取り専用系 | 1項目目が「ファイルの書き込み・編集（読み取り専用）」（テンプレートは「ファイルの作成・編集（Write / Edit ツールの使用禁止）」）。また `git` の項目表現が異なる |
| gha-test | 検証用 | 「やらないこと」セクション自体が存在しない |

**評価者判断**: これらの差異は成功基準の達成に影響しない。テンプレートは新規作成・改修時の基準であり、既存スキルへの遡及適用は本施策のスコープ外。ただし gha-test に「やらないこと」セクションが欠如している点は、テスト用とはいえ安全性の観点から今後の改善候補となる。

#### 修正提案: gha-test への「やらないこと」セクション追加

即時適用可能の3条件を検証:
1. **スコープ内**: gha-test/SKILL.md は本施策の T-002 で frontmatter 変更済みのファイル -- スコープ内
2. **内容明確**: セクション7.7 に検証用テンプレートが定義済み -- 内容明確
3. **競合なし**: gha-test の修正を含む他施策は確認されない -- 競合なし

**提案内容**: `gha-test/SKILL.md` のステップ6の前に以下を追加する:

```markdown
## やらないこと
- `git rebase`, `git reset --hard` 等の破壊的 git 操作
- 本番データ（`sessions/initiatives/` 内の実施策）の変更
```
