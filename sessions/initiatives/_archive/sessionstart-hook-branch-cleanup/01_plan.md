# 実施計画: SessionStart hook ブランチ整理自動化

## フェーズ構成

| フェーズ | 内容 | 成功基準 |
|---------|------|----------|
| A | L1: 提案・計画・タスク作成 | 00〜02 が完成し L2 にディスパッチ可能 |
| B | L2-worker: スクリプト作成・設定更新・スキル更新 | 全タスク完了・コミット済み |
| C | L2-evaluator: スクリプト構文検証・設定整合性検証 | 評価レポート完成 |
| D | L1: ゲート判定 | 通過 / 条件付き通過 / 差し戻し |

## 成功基準（全体）

1. `.claude/hooks/session-start-branch-cleanup.sh` が `bash -n` で構文エラーなし
2. main ブランチ時: `git pull` のみ実行される
3. 別ブランチ時: push → PR 確認/作成 → main 切替フローが実行される
4. gh 不在時: PR ステップがスキップされ、warnings に記録される
5. 全マネージャースキル（6ファイル）が一貫して hook 自動実行の記述に更新されている
6. `docs/workflow.md` が hook 自動化を反映している
7. `.claude/rules/session-start-branch-cleanup.md` が設計仕様として再定義されている

## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| hook スクリプトの構文エラー | 中 | 高 | `bash -n` による構文チェックを評価計画に含める |
| settings.json の JSON 構文エラー | 低 | 高 | evaluator が JSON バリデーション実施 |
| マネージャースキル間の記述不整合 | 中 | 中 | 共通テキストをタスクで明示し、6ファイル統一を検証 |
| hook 実行時の exit code 問題 | 中 | 高 | 全パスで exit 0 を保証する設計 |
