# 実施計画: CSV コンフリクト防止の仕組み構築

## フェーズ構成

| フェーズ | 内容 | 成功基準 |
|---------|------|----------|
| 1 | スクリプト作成・テンプレート拡張 | SC-1, SC-2 |
| 2 | 既存データ移行・ラウンドトリップ検証 | SC-3, SC-4 |
| 3 | スキルファイル更新・hook 設定・ドキュメント更新 | SC-5, SC-6, SC-7 |

## 成功基準（全体）

- **SC-1**: `scripts/generate-csvs.py` が `issues/entries/` と `backlog/entries/` から CSV を正しく生成できる
- **SC-2**: `issues/_template.md` に「対応・クローズ情報」セクションが追加されている
- **SC-3**: 既存 CSV の全行に対応する .md ファイルが `issues/entries/` に存在する（約50件の新規生成）
- **SC-4**: ラウンドトリップテスト合格: 移行後の .md → CSV 再生成 → 元 CSV のデータと一致（列順・内容）
- **SC-5**: `課題管理.csv` または `backlog.csv` を参照する全スキルファイル（25件）で「CSV 直接編集」指示が「.md 編集（CSV 自動生成）」に更新されている
- **SC-6**: PreCommit hook が `.claude/settings.json` に設定され、コミット時に CSV が自動再生成される
- **SC-7**: `docs/workflow.md`、`issues/README.md`、`backlog/README.md` が更新されている

## リソース・前提条件

- Python 3 が実行環境で利用可能であること
- Claude Code の hooks 機能（PreCommit）が利用可能であること

## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| 移行時のデータ損失 | 低 | 高 | ラウンドトリップテストで検証。git 履歴が復元手段 |
| .md パース失敗 | 中 | 中 | エラー報告機能。パース不可はスキップ＋警告 |
| スキル更新漏れ | 中 | 中 | grep で全参照を網羅確認 |
| 旧フォーマット backlog | 低 | 低 | 移行スクリプトでテーブル形式に変換 |
