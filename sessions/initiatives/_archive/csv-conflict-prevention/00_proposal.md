# 改善施策提案: CSV コンフリクト防止の仕組み構築

## 背景・課題

`プロセス改善_課題管理.csv` は複数セッション（L1施策・トリアージ等）が並行して行を追加するため、マージ時にコンフリクトが頻発している。直近では ISS-059 の ID 衝突が発生し、リナンバー＋参照更新の手作業が必要になった。

CSV は行単位の追記が主な操作だが、git のマージは同一ファイル末尾への追記を競合として扱うため、並行作業と本質的に相性が悪い。`backlog/backlog.csv` も同様の構造的問題を持つ。

現状の課題管理 CSV は 64行15列あるが、対応する `issues/entries/ISS-XXX.md` は13件のみで、約50件は CSV にしかデータが存在しない。さらに CSV には .md に存在しない4列（対応方針_確定、確定日、タスク化先、備考）がある。

## 目標

- CSV マージコンフリクトの発生をゼロにする（原理的に回避）
- コンフリクト解消にかかる手作業（リナンバー・参照更新）を排除する
- 並行セッション間での ID 衝突を防止する
- `.md` ファイルを正の情報源（Source of Truth）として一元化する

## スコープ

### やること
- `プロセス改善_課題管理.csv` + `issues/entries/ISS-XXX.md` の完全移行
- `backlog/backlog.csv` + `backlog/entries/*.md` の完全移行
- CSV 自動生成スクリプト（Python）の作成
- 既存 CSV-only データ約50件の .md ファイル自動生成（移行スクリプト）
- 旧フォーマット backlog エントリ 13件のテーブル形式変換
- issues テンプレートへの CSV-only 4列の新セクション追加
- PreCommit hook による CSV 自動再生成の設定
- スキルファイル 25件 + テンプレート 1件の「CSV 直接編集」指示の更新
- ドキュメント（README、workflow.md）の更新

### やらないこと
- `automation-candidates/` 等の他 CSV への適用（別施策で検討）
- CSV ファイルの廃止（自動生成で維持。人間の俯瞰用途として有用）

## 期待される効果

- CSV マージコンフリクト: 頻発 → ゼロ（個別 .md ファイルは並行編集しても競合しない）
- ID 衝突リスク: あり → なし（ファイル名ベースの ID 管理）
- 手動 CSV 編集: 必要 → 不要（.md 編集のみで CSV は自動生成）
- データの一元性: CSV と .md の二重管理 → .md が唯一の Source of Truth

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 移行時のデータ損失 | 高 | ラウンドトリップテストで検証。git 履歴が復元手段 |
| .md パース失敗（不正なフォーマット） | 中 | スクリプトにエラー報告。パース不可ファイルはスキップ＋警告 |
| スキル 25件の更新漏れ | 中 | grep で全参照を網羅確認。評価ワーカーのチェック |
| 旧フォーマット backlog のパース失敗 | 低 | 13件は移行スクリプトでテーブル形式に変換 |

## 壁打ちの背景

### アプローチの選択
バックログに4つのアプローチ案が記載されていた:
- (A) 個別 .md ファイルを正の情報源とし CSV を自動生成 ← **採用**
- (B) ID 採番のプレフィクス分離のみ
- (C) マージ前に CSV を自動ソート/再生成する git hook
- (D) CSV を廃止

(A) を選択した理由: 根本的な解決であり、個別ファイルは並行編集しても競合しない。CSV は人間の俯瞰用途として自動生成で維持する。

### CSV-only 4列の移行判断
課題管理 CSV に .md テンプレートにない4列（対応方針_確定、確定日、タスク化先、備考）が存在することが調査で判明。これらを .md テンプレートの新セクション「対応・クローズ情報」として統合する方針を決定。

### 既存データの移行方針
約50件の CSV-only 課題について「全件 .md 作成」を選択。移行スクリプトで自動生成し、完全移行とする。

## 備考・設計パターン

- 生成スクリプトは Python（stdlib のみ、外部依存なし）で実装
- CSV の列順は既存と同一にし後方互換性を維持
- ID 昇順ソートで決定論的出力を保証
- PreCommit hook で自動実行、手動ステップ不要
