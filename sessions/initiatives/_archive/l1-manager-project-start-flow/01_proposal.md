# 改善施策提案: l1-manager-project-start-flow

## 背景・課題
l1-manager SKILL.md の入力パターンは「backlog: <施策名>」（新規）と「施策: <パス>」（継続）の2種。projects/ ディレクトリでプロジェクト単位の施策管理を導入したが、project から次の施策を開始するフローが l1-manager に存在しない。

## 目標
l1-manager SKILL.md に「project: <プロジェクト名>」入力パターンを追加し、project の todo.md から次の施策を選択・開始できるようにする。

## スコープ
### やること
- SKILL.md の「セッション開始時の入力解釈」にパターン3を追加
- SKILL.md の「セッションライフサイクル todo 登録」にパターン3を追加
- docs/workflow.md の該当セクションを更新（可視化文書の連動更新）

### やらないこと
- project-progress-sync スキルの変更
- project テンプレートの変更

## 期待される効果
- project 単位での施策管理と l1-manager の入力フローが接続される
- 「次に何をやるか」を project の todo.md で把握してそのまま施策開始できる

## リスク
| リスク | 影響度 | 対策 |
|--------|--------|------|
| 既存パターンとの競合 | 低 | パターン3は独立した新規追加のため競合なし |

## 壁打ちの背景
ai-dev-chain プロジェクトの管理ディレクトリ作成時に導線の欠如に気づいた。小規模変更のため即時施策として実行。
