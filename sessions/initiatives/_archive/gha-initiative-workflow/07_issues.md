### [2026-03-14] issue_comment トリガーの PR コメント発火問題
- **種別**: 知見
- **内容**: GHA の `issue_comment` トリガーは PR コメントでも発火する。Initiative ワークフローでは Issue コメントのみに反応すべきなので、`github.event.issue.pull_request == null` フィルタを追加した。今後 `issue_comment` トリガーを使うワークフローでは同様の考慮が必要
- **対応方針（案）**: gha-guideline に注意事項として記載を検討（当施策では対応済み）
- **関連タスク**: T-002
[転記不要 — 当施策内で対応済み。gha-guideline への追記は別施策として検討]

### [2026-03-14] wallbash.yml の `$(cat)` シェル展開問題（評価者発見）
- **種別**: 不具合
- **内容**: `initiative-wallbash.yml` のプロンプト内で `$(cat /tmp/issue_body.txt)` を使用しているが、GHA Action の `prompt:` 入力フィールドではシェルコマンド置換は展開されない。Issue 本文がプロンプトにインライン注入されず、リテラル文字列として渡される。Claude が `Bash(cat *)` 権限で自発的にファイルを読む可能性はあるが、意図した動作ではない
- **対応方針（案）**: `${{ }}` 式で Issue 本文を注入するか、プロンプトで明示的にファイル読み取りを指示する
- **影響**: 壁打ちフェーズで Issue 本文の内容が Claude に伝わらず、提案の品質が低下する可能性
[転記不要 — 当施策内の成果物修正で対応可能]

### [2026-03-14] execute/close.yml の不要な `Bash(cat *)` 権限（評価者発見）
- **種別**: 改善提案
- **内容**: `initiative-execute.yml` と `initiative-close.yml` の Claude Code 権限に `Bash(cat *)` が含まれているが、これらのワークフローでは使用されない。最小権限の原則に基づけば除外すべき
- **対応方針（案）**: execute.yml と close.yml の settings.permissions.allow から `Bash(cat *)` を除外
- **影響**: 低（`Read` ツールで代替可能なため実害は小さい）
[転記不要 — 当施策内の成果物修正で対応可能]
