### E-002 詳細: allow リスト突合

**triage.md の操作を列挙し、必要なツールを導出**:

| triage.md の操作 | 必要な Claude Code ツール | allowedTools に含まれるか |
|-----------------|------------------------|------------------------|
| テンプレートファイルのコピー（ステップ1, 4） | Bash(cp *) | ✅ |
| ディレクトリ作成（ステップ1, 4） | Bash(mkdir *) | ✅ |
| ファイル読み取り（ステップ2: inbox, backlog, CSV, skills 等） | Read | ✅ |
| ファイル検索（ステップ2: inbox/*.md, backlog/entries/ 等） | Glob | ✅ |
| コンテンツ検索（ステップ2: ルール整合性チェック等） | Grep | ✅ |
| ファイル作成・書き込み（ステップ3-9: plan, report, log 等） | Write | ✅ |
| ファイル編集（ステップ4: backlog.csv 更新等） | Edit | ✅ |
| ファイル削除（ステップ4: 処理済み inbox アイテム） | Bash(rm *) | ✅ |
| ディレクトリ一覧（ステップ2: sessions/triage/ 確認等） | Bash(ls *) | ✅ |

**暗黙的コマンドの検証**（L2-worker 申し送りの重点確認事項）:

| コマンド | 必要性 | 理由 |
|---------|--------|------|
| Bash(cat *) | 不要 | Read ツールでファイル読み取りが可能 |
| Bash(date *) | 不要 | 日付はワークフローのユーザーメッセージで `$(date -u +%Y-%m-%d)` として注入済み |
| Bash(find *) | 不要 | Glob ツールでファイル検索が可能 |
| Bash(wc *) | 不要 | Grep の count モードやファイル一覧の手動カウントで代替可能 |
| Bash(head *) | 不要 | Read ツールの offset/limit パラメータで代替可能 |
| Bash(tail *) | 不要 | Read ツールの offset パラメータで代替可能 |
| Bash(echo *) | 不要 | Write ツールでファイル作成が可能。デバッグ出力は不要 |

**結論**: 過不足なし。allowedTools の9エントリで triage.md の全操作をカバーしている。

### E-003 詳細: 3層整合性チェック

**対象**: daily-triage.yml

| チェック | 内容 | 結果 | 根拠 |
|---------|------|------|------|
| L1 ⊇ L3 | GHA permissions がプロンプト指示の GitHub API 操作をカバー | ✅ | Claude はプロンプトで git/gh 操作を指示されていない（「コミット不要」明記）。Shell ステップが git push（contents:write）と gh pr create（pull-requests:write）を実行。両権限あり |
| L2 ⊇ L3 | allowedTools がプロンプト指示の全操作をカバー | ✅ | E-002 の突合結果により確認済み |
| L2 ⊆ 最小権限 | allowedTools が必要最小限 | ✅ | Bash(git *), Bash(gh *) が正しく除外されている。含まれる9ツールはすべて triage.md の操作に必要 |

**設計判断の妥当性**: daily-triage.yml は git commit/push/PR 作成がすべて固定パターン（Shell ステップで実行）であり、Claude への git/gh 委任は不要。SKILL.md セクション2の判断フローに沿った正しい設計である。
