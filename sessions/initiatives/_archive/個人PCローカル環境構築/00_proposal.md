# 改善施策提案: 個人PCローカル環境構築

## 背景・課題

現在 Claude Code Web 環境で開発を行っているが、以下の2つの制約により開発がブロックされている。

1. **ISS-027（1セッション=1リポジトリ制約）**: Web 環境では1セッションにつき1リポジトリしか操作できない。リポジトリ分離施策（dev-process-improvement を別リポに分離）の実行や、マルチリポを横断した作業ができない。
2. **ISS-026（gh CLI 未インストール）**: Web 環境に gh CLI がインストールされておらず、PR 作成や GitHub API 操作が自動化できない。フォールバック（URL手動アクセス）で対応しているが効率が悪い。

ローカル PC（WSL）に Claude Code CLI 環境を構築することで、これらの制約を根本的に解消する。

## 目標

ローカル WSL 環境において以下が動作する状態を構築する。

1. Claude Code CLI が動作し、`--add-dir` によるマルチリポセッションが起動可能
2. gh CLI が動作し、`gh pr create` 等の GitHub 操作が CLI から実行可能
3. CodeSail により iPhone からセッションへのチャット指示が可能
4. VS Code（Remote-WSL）から手動編集が可能

## スコープ

### やること

- WSL（Ubuntu）上への Claude Code CLI 環境構築手順の整理
- gh CLI のインストール・認証手順の整理
- マルチリポ構成の設計（`--add-dir` の使い方、軸リポの決め方）
- セッション起動スクリプトの作成
- CodeSail のインストール・ペアリング手順の整理
- ローカル環境向け CLAUDE.md・settings.json の設計
- 運用ガイド・トラブルシューティングガイドの作成

### やらないこと

- リポジトリ分離施策の実施（本施策はその前提環境の構築のみ）
- CI/CD パイプラインの構築
- Windows ネイティブ環境への対応（WSL のみ）
- 他メンバーへの展開（まず個人環境を構築し、知見を蓄積する）

## 期待される効果

- ISS-026/ISS-027 の解消。リポジトリ分離施策の再開が可能になる
- `--add-dir` によるマルチリポ操作対応で、横断作業の効率が大幅に向上
- gh CLI 利用可能により、PR 作成・レビューのフォールバック対応が不要になる
- iPhone（CodeSail）からの開発指示が可能になり、PC の前にいない時間帯も活用可能

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| WSL 環境と Windows 環境のパス差異による問題 | 中 | WSL 内で完結する構成にし、Windows パスへの依存を避ける |
| CodeSail のサービス安定性（外部依存） | 低 | CodeSail は補助的手段。CLI 直接操作を主とし、CodeSail 不調時も作業継続可能 |
| Claude Code CLI のバージョンアップによる設定変更 | 低 | 設定ファイルにバージョン情報を記載し、アップデート時に差分確認する運用ガイドに含める |

## 壁打ちの背景

L1 マネージャーセッションの壁打ちフェーズで以下を確認・決定した。

1. **CodeSail のスコープ判断**: CLI + マルチリポ構成のみに限定するか、CodeSail（iPhone 連携）も含めるかを確認。→ **一括で含める**と決定。理由: CodeSail の導入は CLI 環境が前提であり、分離すると重複手順が発生するため。
2. **L2-worker のタスクスコープ**: 設定ファイル準備のみか、ドキュメント整備も含むかを確認。→ **ドキュメント整備も含む**と決定。理由: 運用ガイド・トラブルシューティングガイドがないと、環境構築後の日常運用で問題が発生した際に毎回調べ直す必要があるため。
3. **人間タスクとClaude タスクの分離**: この施策は大部分が人間の手作業（WSL導入、CLI インストール等）。L2-worker は設定ファイル設計・起動スクリプト作成・運用ガイド作成に集中する。

## 備考・設計パターン

- `refs/個人PCローカル環境構築-手順書.md` に詳細な手順書（フェーズ2 on WSL 構成）がある。この手順書をベースに、L2-worker タスクと人間タスクを分離して施策化する。
- マルチリポ構成は「軸リポ + `--add-dir`」パターン。軸リポをどちらにするかはタスクごとに判断する運用とする。
- セッション起動スクリプト（`start-workspace.sh`）でリポの組み合わせを定義し、毎回の `--add-dir` 入力を省略する。
