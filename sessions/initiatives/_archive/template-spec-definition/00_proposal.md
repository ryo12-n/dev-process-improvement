# 改善施策提案: template-spec-definition

## 背景・課題

各セッションタイプの `sessions/*/_template/` に「どのファイルが含まれるべきか」の正解定義が一箇所にない。関連情報が session-flow-policy §4.3、session-lifecycle-policy §1.3、各エージェント定義の担当ファイルテーブル、`_template/` 実装に散在している。加えて、session-lifecycle-policy の §1.2（30列マトリクス）と §1.3（8列テーブル）は冗長で、共通パターンが集約されていない。

## 目標

- session-lifecycle-policy §1.2 を 3 アーキタイプに集約（30列 → マネージャー/ワーカー/評価ワーカーの 3 列）
- session-lifecycle-policy §1.3 を 2 パターン + バリアントに集約（8列 → ロール別記録先）
- §1.4 テンプレート構成仕様を新設し、全テンプレートの正解定義を Source of Truth として確立
- SC-009 を追加してテンプレート構成仕様の準拠チェックを自動化

## スコープ
### やること
- §1.2 + §1.3 のリファクタリング（フォーマット変換、推奨→必須の統一）
- §1.4 テンプレート構成仕様の新設（基本形 + バリエーション方式）
- SC-009 チェックターゲットの追加（4ファイル変更）
- SC-006 reference セクションの新マトリクス形式への更新
- テンプレート構造統一改修の backlog エントリ起票

### やらないこと
- テンプレートの物理的なディレクトリ構造の変更（別施策として backlog に起票）
- エージェント定義の担当ファイルテーブルの変更
- 各セッションタイプの SKILL.md のフロー変更

## 期待される効果

- テンプレート構成の正解が一箇所に集約され、新セッション作成・SC チェック時の参照コスト削減
- §1.2 マトリクスが 30 列 → 3 列に集約され可読性向上
- SC-009 による仕様 vs 実態の自動整合チェック

## リスク
| リスク | 影響度 | 対策 |
|--------|--------|------|
| §1.2 集約時の情報欠落 | 中 | 旧30列マトリクスの全セルを新形式にマッピングし差分検証 |
| SC-006 との整合性 | 低 | §1.2 形式変更に伴い reference セクションを更新 |
| 外部参照の破損 | 低 | grep で参照元を洗い出し確認 |

## 壁打ちの背景

- §1.2 のロール列名を「マネージャー」「ワーカー」「評価ワーカー」に統一（ユーザー指示）
- 「推奨」を廃止し「必須」に統一（ユーザー指示）
- 計画ステージは全ロールで「必須」（ユーザー指示）
- §1.4 の基本モデルは multi-phase manager パターン。他パターンはバリエーションとして記述（ユーザー合意）
- テンプレート構造の物理的統一改修は別施策として backlog に起票（ユーザー選択: 仕様定義のみ先行）

## 備考・設計パターン

本施策はフォーマット変換が中心。情報の追加・削除ではなく、既存情報の再構成。変換前後の情報量が同等であることの検証が重要。
