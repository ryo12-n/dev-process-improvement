# 実施計画: backlogテンプレート凡例強化

## フェーズ構成

| フェーズ | 内容 | 成功基準 |
|---------|------|----------|
| 1 | テンプレート強化（`backlog/_template.md` 更新） | 新フィールド・チェックリスト・凡例が追加されている |
| 2 | README への手動チェックリスト追加 | 起票時チェックリストが README に記載されている |
| 3 | 既存エントリの対象リポジトリ欠損補完 | entries/ 内の非標準フォーマットエントリを確認し、欠損を補完する |

## スケジュール

1. `backlog/_template.md` の凡例・フィールド強化
2. `backlog/README.md` の手動チェックリスト追加
3. 欠損エントリの補完（対象リポジトリ値が空または不明なエントリ）

## 成功基準（全体）

1. `backlog/_template.md` に「依存する環境前提」フィールドと「関連エントリ」セクションが追加されている
2. emoji 表記ルール（🔴/🟡/🔵 優先度、🔴/🟡/🟢 ステータス等）が凡例として明記されている
3. 手動チェックリストがテンプレート内に組み込まれている
4. `backlog/README.md` に起票時チェックリストの参照が記載されている
5. 非標準フォーマットエントリ（メタ情報テーブルなし）の対象リポジトリが確認・補完されている

## リソース・前提条件

- `backlog/_template.md` および `backlog/entries/` 配下のファイルを直接編集する
- 既存エントリへの変更は「対象リポジトリ欄の補完」のみに限定し、フォーマット全体の統一は別施策とする
- CI/CD 自動バリデーションは実装しない（スコープ外）

## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| 既存エントリとの互換性が崩れる | 低 | 中 | 新フィールドはオプション扱いで追加。既存エントリの書き換えは欠損補完のみ |
| テンプレート変更が浸透しない | 中 | 中 | README にチェックリストポインターを追記し、起票者が確認できる仕組みにする |
