# 実施計画: setting.json へ allow リスト追加

## フェーズ構成

| フェーズ | 内容 | 成功基準 |
|---------|------|----------|
| 1 | allow ルールの設計・追加 | settings.json に allow ルールが追加され、構文エラーがないこと |

## 成功基準（全体）

1. よくある操作（ファイル操作・git・gh CLI・基本シェルコマンド）で許可プロンプトが出ないこと
2. deny ルール対象（.env, secrets/, *.pem, *.key）へのアクセスは引き続きブロックされること

## リソース・前提条件

- Claude Code の settings.json の `allow`/`deny` の優先順位: deny が allow より優先
