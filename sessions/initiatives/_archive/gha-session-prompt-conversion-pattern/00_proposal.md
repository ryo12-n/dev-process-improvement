# 改善施策提案: GHA セッションプロンプト変換パターンガイド

## 背景・課題

SKILL.md から GHA ワークフロープロンプトへの変換パターンがガイドライン化されていない。現在5つのGHAスキル（wallbash, execute, close, question, test）が実装されているが、変換パターンは暗黙知として各実装に散在している。新セッションタイプのGHA化のたびに既存実装を参照・模倣する必要があり、効率が悪い。

関連する知見は以下に分散している:
- `gha-guideline/SKILL.md` — 権限3層モデル・ワークフロー権限マトリクス
- `refs/claude-code-action/knowledge.md` — 実装知見21件
- 各GHAスキルの SKILL.md と対応する workflow YAML

## 目標

`gha-guideline/SKILL.md` にセクション7「セッションプロンプト変換パターン」を追加し、CLI セッションスキルを GHA スキルに変換する手順を体系化する。

## スコープ
### やること
- `gha-guideline/SKILL.md` にセクション7を追加（変換全体フロー、SKILL.md テンプレート、プロンプトテンプレート、後続 Shell ステップ設計、アンチパターン）
- frontmatter の description を更新
- 関連ファイル一覧を更新
- `docs/workflow.md` の可視化文書を連動更新

### やらないこと
- 既存 GHA SKILL.md（wallbash/execute/close/question/test）の修正
- workflow YAML の変更
- knowledge.md の変更
- 新規ファイルの作成（既存ファイルへの追記のみ）

## 期待される効果

- 新セッションタイプの GHA 化時に、変換手順を参照できるガイドが存在する
- 暗黙知の形式知化により、変換品質のばらつきを防止
- L2-worker への変換タスク指示が明確化される

## リスク
| リスク | 影響度 | 対策 |
|--------|--------|------|
| gha-guideline/SKILL.md の肥大化 | 低 | 現在292行に対し80-100行の追加で~370-390行。オンデマンド参照文書として許容範囲 |
| 既存実装との乖離 | 低 | 既存5スキルから共通パターンを抽出するため、乖離リスクは小さい |

## 壁打ちの背景

- **配置先の判断**: 新規スキル `gha-session-prompt-conversion-pattern/SKILL.md` として分離する案も検討したが、ユーザー判断で `gha-guideline/SKILL.md` に統合する方針とした。権限設計と変換パターンは密接に関連するため、1ファイルにまとめることで参照効率が向上する
- **対象読者**: L1 マネージャーが GHA 化施策を計画する際の参照資料。L2-worker が実際の変換作業を行う際のタスク指示書としても使える粒度で作成する

## 備考・設計パターン

既存の5つのGHAスキル実装（特に `gha-wallbash/SKILL.md` + `initiative-wallbash.yml`）を参照実装として、共通パターンを抽出・テンプレート化するアプローチを取る。
