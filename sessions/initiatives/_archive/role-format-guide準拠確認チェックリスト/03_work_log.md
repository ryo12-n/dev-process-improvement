#### 要件一覧

**A. 必須セクション（4つ）**
1. `## あなたの役割` — ロールが何をするかを1〜2文で説明する
2. `## 作業フロー` — やるべき作業を番号付きステップで列挙する。各ステップには必要な詳細を含める
3. `## やること` — このロールが担う責任を箇条書きで明示する
4. `## やらないこと` — このロールの範囲外の作業を箇条書きで明示する。担当ロールがある場合は明記する

**B. 任意セクション（2つ）**
5. `## 担当ファイル` — テーブル形式（ファイル | 操作）で記載する
6. `## 停止ルール` — 作業を中断すべき条件を記載する

**C. セクション順序**
```
1. あなたの役割（必須）
2. 作業フロー（必須）
3. やること（必須）
4. やらないこと（必須）
5. 担当ファイル（任意）
6. 停止ルール（任意）
7. ロール固有の追加セクション（任意）
```

**D. YAML フロントマター**
- `paths:` バインドを使う場合のみ追加する
- 不要なら省略してよい
- 形式: `---\npaths: "glob パターン"\n---`

**E. セクション見出しのレベル**
- 必須・任意セクションの見出しは `##`（H2）レベル

**F. 「やらないこと」の書き方**
- 担当ロールがある場合は明記する（例: `→ {担当ロール} が担当`）

**成果物**: 上記の要件一覧（本ログ内に記録）
