# 計画レポート: ai-dev-chain-gen-req — Plan-Worker Set-1

## サマリ

gen-req SKILL.md のプロンプト実装を中心に、3タスク・2 Wave の計画を策定した。TODO 5項目は相互依存が強いため1タスク（T-002）に統合し、テスト用プロジェクト作成（T-001）を並列実行可能な独立タスクとして分離、動作テスト（T-003）を依存 Wave に配置した。Wave 1 で最大2並列、Wave 2 は1ワーカーの構成。

## 所感・次フェーズへの申し送り

- T-002 が本施策の主要タスク。プロンプト実装の品質が成果物全体の品質を決定するため、ワーカーは `request.md` の8セクションと `requirements.json` の6セクションの対応関係を意識して実装すること
- T-003 の動作テストでは、生成された JSON のスキーマ準拠確認と、overview.md の入出力仕様との整合確認を必ず実施すること
- 本施策で確立するプロンプト設計パターン（入力読み取り→分析→JSON 生成→MD 生成→レポート）は M4 以降の gen-design 等でも参照される。パターンの再利用性を意識した実装が望ましい
