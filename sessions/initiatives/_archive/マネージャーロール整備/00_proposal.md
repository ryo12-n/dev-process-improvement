# 改善施策提案: マネージャーロール整備

## 背景・課題

ai-driven-dev-patterns の `roles/dev_manager.md` は「コード開発ワークフロー・ロール定義の実装」施策で新規作成された開発タスクオーケストレーターである。現在は以下の制約があり、汎用的なマネージャーロールとして使える状態ではない。

1. **知見収集の仕組みがない**: 開発セッションで得られた知見（ルール化候補・参考情報）を抽出・記録する仕組みがない。学びが蓄積されず、同じ問題を繰り返す
2. **フェーズゲートチェックが形式化されていない**: reviewer の合否判定のみで、完了条件の体系的な検証や3択判定（通過/条件付き通過/差し戻し）がない
3. **進捗管理の構造がない**: log.md が追記ログのみで、タスク完了率やブロッカーの可視化ができない
4. **課題管理フローがない**: issues.md は存在するが、課題の重大度判定や集約・棚卸しのフローが未定義
5. **dispatcher 依存**: ユーザーとの対話がすべて dispatcher 経由。dev_manager を直接起動して使うことができない

## 目標

dev_manager を上記5つの責務を持つ汎用マネージャーロールに進化させ、各シナリオで直接ユーザーと対話しながら開発タスクをオーケストレーションできるようにする。

## スコープ

### やること

- `roles/dev_manager.md` に5つの責務（知見収集・ゲートチェック・進捗管理・課題管理・人間IF一元化）を追加
- dispatcher 依存を完全に除去し、ユーザーとの直接対話フローに変更
- 知見収集モデルとして dev-process-improvement の L1/L2 パターン（ルール化候補・参考情報テーブル必須）を採用
- `roles/_base/common.md` のアーキテクチャ図・参照を更新
- `sessions/_template/` の report.md, plan.md, log.md, issues.md を更新

### やらないこと

- 個別専門ロール（feature_builder.md, test_writer.md 等）の直接編集（_base/common.md 経由で間接反映）
- 設計書（docs/design/dev-workflow-detail.md 等）の更新（ISS-024 は別施策で対応）
- dispatcher スキル自体の変更（dev_manager 側の依存除去のみ）

## 期待される効果

- **再利用性向上**: dev_manager が dispatcher なしで直接起動可能になり、各シナリオで汎用的に活用可能
- **人間IF一元化**: 人間とのインターフェースが dev_manager に集約され、他ロールの自律性が向上
- **計画品質向上**: 構造化された完了条件・成功基準・リスク評価により、セッション計画の品質が向上
- **知見の蓄積**: 開発セッションごとに知見が記録・ルーティングされ、組織学習が促進
- **品質ゲートの確実化**: 3択判定による形式化されたゲートチェックで見落としを防止

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| dispatcher 依存除去により既存フローが壊れる | 中 | dev_manager.md と _base/common.md のみ変更。dispatcher スキル自体は変更しない |
| 5項目の同時変更でスコープが大きくなりすぎる | 中 | タスクを独立性高く分割し、各タスクの完了条件を明確に定義 |
| sessions/_template/ の変更が既存セッションと不整合 | 低 | テンプレート変更は追加的（既存セクション削除なし） |

## 壁打ちの背景

壁打ちフェーズで以下の方針を確認した。

- **スコープ**: 全5項目を1施策で実施する。優先度の高い項目のみに絞る案もあったが、ユーザーの判断で全項目を実施
- **人間IF**: 「両方対応（dispatcher あり/なし）」案と「直接対話に完全移行」案があり、ユーザーの判断で **直接対話に完全移行** を選択
- **知見収集モデル**: dev-process-improvement の L1/L2 モデルと同じパターン（ルール化候補・参考情報テーブル必須）を採用

## 備考・設計パターン

- dev-process-improvement の L1/L2 モデル（知見テーブル、ゲートレビュー3択判定）を参考にするが、コード開発コンテキスト向けに適応する
- 知見のルーティング先: `roles/` or `.claude/rules/` / `docs/collab-log.md` / `issues.md` / なし
