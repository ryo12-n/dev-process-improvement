# 改善施策提案: Claude Code ビルトイン skills 評価・取り込み

## 背景・課題

Claude Code が標準で提供するビルトイン skills（/simplify, /batch, /debug, /loop, /claude-api）について、dev-process-improvement プロジェクトでの有用性評価と活用判断が行われていない。どの skills がプロジェクトのワークフローに適用可能かが不明確で、有用な skills の採用が遅れている。

## 目標

全5つのビルトイン skills を体系的に評価し、プロジェクトでの活用方針を明確化する。評価結果を `.claude/skills/` にオンデマンド参照スキルとして配置し、Claude が必要時に自動参照できるようにする。

## スコープ
### やること
- 5つのビルトイン skills の機能概要・適用可能性・具体的利用シーンの評価
- `.claude/skills/builtin-skills-reference/SKILL.md` の作成（評価結果を含むオンデマンド参照スキル）
- `docs/workflow.md` への連動更新（可視化文書）

### やらないこと
- ビルトイン skills のカスタム拡張実装
- コミュニティ・外部 skills の評価（ビルトインのみが対象）
- ai-driven-dev-patterns リポジトリへの変更

## 外部リポジトリ

| リポジトリ名 | ローカルパス | 変更内容の概要 |
|-------------|------------|-------------|
| なし | - | - |

## 期待される効果

- ビルトイン skills の活用判断基盤の確立
- 有用な skills の早期採用によるプロジェクト効率向上
- 新規 skills 追加時の評価基準の明確化

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| ビルトイン skills の仕様がバージョンで変わる可能性 | 低 | 評価日を明記し、定期的な再評価を推奨事項として記載 |
| Markdown 文書主体のプロジェクトでコード向け skills の適用場面が限定的 | 低 | 「適用場面なし」も正当な評価結果として記録 |

## 壁打ちの背景

- **対象リポジトリの変更**: バックログでは ai-driven-dev-patterns が対象だったが、ユーザー指示により dev-process-improvement に変更。プロセス改善リポジトリ自体の skills 活用を優先する
- **成果物の形式**: docs/ 運用ガイドと skills オンデマンド参照の2案を検討。将来的に skills 統合予定であることを踏まえ、最初から `.claude/skills/` に配置する方針を採用。`user-invocable: false` + 適切な `description` で必要時のみ自動参照されるため、コンテキストコストの懸念も解消
- **ビルトイン skills のディレクトリ作成**: ビルトイン skills は Claude Code に同梱されており `.claude/skills/` への配置は不要。本施策で作成するのは「評価結果のリファレンスドキュメント」としてのスキルファイル

## 備考・設計パターン

本施策はコード変更ではなく調査・評価・文書化が中心の小規模施策。単一フェーズで完結する。
