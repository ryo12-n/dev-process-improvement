# 実施計画: AI補助自動化ツール検討

## フェーズ構成

| フェーズ | 内容 | 期間 | 成功基準 |
|---------|------|------|----------|
| 1 | 定型作業の洗い出し・自動化候補一覧の作成・費用対効果評価・ロードマップ策定 | 1セッション | 一覧表に10件以上の候補が記載され、各候補に評価軸（頻度・実装難易度・手動コスト・自動化手段）が記入されている。ロードマップが策定されている |

## スケジュール

1. 定型作業の網羅的洗い出し（skills/rules/agents 定義の走査）
2. 各定型作業の自動化可能性の調査（Claude Code hooks, MCP tools 等）
3. 費用対効果の評価表作成
4. 優先度付きロードマップの策定

## 成功基準（全体）

1. 自動化候補一覧に10件以上の定型作業が記載されている
2. 各候補に対して評価軸（頻度・実装難易度・手動コスト・自動化手段）が記入されている
3. 優先度付きロードマップが策定され、上位3件の候補について具体的な自動化手段が提案されている

## リソース・前提条件

- `.claude/skills/` 配下のスキル・エージェント定義ファイル
- `.claude/rules/` 配下のルールファイル
- `sessions/initiatives/_archive/` の過去施策（作業パターンの参考）
- Claude Code の hooks / MCP tools / settings に関するドキュメント

## リスクと対策

| リスク | 発生確率 | 影響 | 対策 |
|--------|---------|------|------|
| 調査対象が多すぎて1セッションで完了しない | 中 | 中 | 主要なスキル定義（l1-manager, triage-manager）に重点を置き、網羅性より実用性を優先する |
| Claude Code hooks の制約により自動化手段が限定される | 低 | 低 | 制約は一覧に明記し、代替手段（シェルスクリプト等）も検討する |
