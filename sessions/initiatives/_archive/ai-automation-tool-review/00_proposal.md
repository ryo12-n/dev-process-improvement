# 改善施策提案: AI補助自動化ツール検討

## 背景・課題

AI を活用した作業補助において、手動で行っている定型作業の中に自動化可能な候補が存在する。現状ではどの作業が自動化に適しているかの一覧化・評価が行われておらず、効率化の機会を逃している可能性がある。

具体的には、以下のような定型作業が毎セッション手動で実施されている:
- セッション開始時のブランチ整理（7ステップの手順）
- テンプレートコピー＋施策ファイル群の作成
- 作業履歴のタイムスタンプ付き記録
- 知見記録（ルール化候補・参考情報テーブル）
- 課題起票・CSV転記（07_issues.md → CSV → ISS-XXX.md）
- コミットメッセージ規約の遵守
- 施策クローズ手順（アーカイブ移動・PR作成・知見ルーティング）
- トリアージ走査（inbox/backlog/CSV の定期チェック）

## 目標

1. 現在の開発プロセスにおける定型作業を網羅的に洗い出し、自動化候補一覧を作成する
2. 各候補の費用対効果（実装コスト vs 削減効果）を評価する
3. 優先度の高い自動化候補から段階的に導入できるロードマップを策定する

## スコープ
### やること
- プロセス内の定型作業の網羅的な洗い出し（skills/rules/agents 定義、過去施策の作業パターンから抽出）
- 各定型作業に対する自動化手段の調査（Claude Code hooks, MCP tools, custom slash commands, シェルスクリプト等）
- 費用対効果の評価（実装難易度・頻度・手動作業時間の観点）
- 優先度付きロードマップの策定

### やらないこと
- 自動化ツールの実装（PoC 含む）— 次施策として個別に起票する
- 外部 CI/CD（GitHub Actions 等）の導入検討 — 本施策は Claude Code 環境内で完結する自動化に焦点を絞る
- ai-driven-dev-patterns リポジトリ固有の作業自動化

## 期待される効果

- 自動化候補の可視化により、改善の優先順位付けが可能になる
- 手動作業の負荷軽減により、セッションあたりの生産性が向上する
- ロードマップにより、段階的・計画的な自動化導入が実現できる

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 調査範囲が広すぎて施策が肥大化する | 中 | スコープを「一覧化＋評価＋ロードマップ」に限定し、実装は別施策とする |
| Claude Code の自動化機能（hooks 等）の制約で実現不可能な候補が多い | 低 | 調査段階で実現可能性を確認し、一覧に明記する |

## 壁打ちの背景

- スコープは「プロセス内の定型作業全般」を対象とすることで合意
- 成果物は「一覧化＋ロードマップ」（バックログ原文どおり）
- PoC 実施は含めず、調査・評価・ロードマップ策定までとする
- CI/CD 等の外部ツール連携は対象外とし、Claude Code 環境内の自動化に焦点を絞る

## 備考・設計パターン

本施策は調査・分析が主体のため、成果物は主にドキュメント（一覧表・評価表・ロードマップ）となる。
自動化候補の評価軸として以下を使用する:
- **頻度**: 毎セッション / 毎施策 / 不定期
- **実装難易度**: 高（新規ツール開発） / 中（既存機能の組み合わせ） / 低（設定変更のみ）
- **手動コスト**: 高（複数ステップ・ミスしやすい） / 中 / 低
- **自動化手段**: Claude Code hooks / MCP tools / シェルスクリプト / skills 拡張
