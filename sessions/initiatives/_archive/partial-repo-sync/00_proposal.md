# 改善施策提案: 2つのリポジトリの部分的同期

## 背景・課題

dev-process-improvement リポジトリのディレクトリ構成に以下の課題がある。

1. **作業履歴の分散**: `triage/` と `initiatives/` に作業履歴が分散しており、統一的な管理構造がない
2. **triage のアーカイブ不在**: initiatives には `_archive/` があるが、triage には完了セッションの整理場所がなく、17個以上のセッションディレクトリが並列に存在している
3. **空テンプレートのノイズ**: triage セッション作成時に `triage/_template/` を丸ごとコピーするため、`workers/_template/` がセッションフォルダ内に残り、PRの差分にノイズとして現れる

また、将来的に ai-driven-dev-patterns にも同等の initiative/triage セッション構成を導入する際、2リポジトリ間での同期を容易にするための観点整備が必要。

## 目標

1. `sessions/` ディレクトリを導入し、triage と initiatives の作業履歴を統一管理する
2. triage に `_archive/` を用意し、完了済みセッションを整理可能にする
3. triage セッション作成時に空テンプレートがコミットされないようコピー方法を改善する
4. ai-driven-dev-patterns への同期に向けた構成整備とチェックリスト作成

## スコープ

### やること
- `sessions/` ディレクトリ導入（`sessions/triage/`, `sessions/initiatives/` 構成）
- 既存の `triage/`, `initiatives/` を `sessions/` 配下に移動
- `sessions/triage/_archive/` の用意
- triage-manager スキル定義のテンプレートコピー手順を改善（`workers/_template/` のコピー回避）
- CLAUDE.md のディレクトリ構成テーブル更新
- ai-driven-dev-patterns への同期チェックリスト作成
- 関連するスキル定義・ルールファイル・ドキュメントのパス更新

### やらないこと
- 既存の完了済みトリアージセッションのアーカイブ移動（構成のみ用意し、移動は後続で行う）
- ai-driven-dev-patterns への実際の同期実施（チェックリスト作成まで）
- initiatives テンプレートの内容変更（パス移動のみ）

## 期待される効果

- PR差分から空テンプレートのノイズが解消される
- triage の完了セッション整理が可能になる
- 2リポジトリ間のプロセス同期の基盤が整う

## リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| パス移動に伴うスキル定義・ルールの参照切れ | 高 | 全参照箇所を事前に洗い出し、一括更新する |
| 移動後の git 履歴の追跡性低下 | 中 | `git mv` を使い、コミットメッセージで移動元を明記する |
| ai-driven-dev-patterns 側の構成差異 | 低 | チェックリストで差異を管理し、段階的に同期する |

## 壁打ちの背景

- sessions ディレクトリは `sessions/triage/`, `sessions/initiatives/` のように既存ディレクトリを配下に移動する構成で合意
- ai-driven-dev-patterns への同期は本施策内でチェックリストまで整備する（実際の同期実施は後続施策「リポジトリ間テンプレート同期の検証チェックリスト」と連携）
- 空テンプレートの問題は、`triage/_template/` コピー時に `workers/_template/` が含まれ、PRの差分にノイズとなる点。実害はないが無駄なファイル生成が気になる
- CLAUDE.md のディレクトリ構成テーブル更新も本施策のスコープに含める
