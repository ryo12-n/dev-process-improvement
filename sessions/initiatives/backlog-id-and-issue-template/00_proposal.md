# 改善施策提案: backlog-id-and-issue-template

## 背景・課題

- backlog.csv に一意の ID がなく、施策名（日本語テキスト）が実質的な識別子になっている
- ワークフロー（backlog-to-issue）で施策を指定する際、長い日本語名を正確に入力する必要がある
- backlog → issue 起票時にテンプレートがなく、転記内容がワークフローのシェルスクリプトにハードコードされている

## 目標

- backlog エントリに BL-001 形式の一意 ID を導入し、ID で施策を指定できるようにする
- backlog-to-issue ワークフローの入力を BL-xxx ID に変更する
- backlog の内容をそのまま issue に転記する方式を整備する

## スコープ
### やること
- backlog.csv に ID 列を追加し、既存53件に BL-001〜BL-053 を採番
- backlog/_template.md にバックログ ID フィールドを追加
- backlog/entries/*.md のメタ情報に BL-xxx ID を追記
- backlog-to-issue.yml のワークフロー入力を BL-xxx ID 指定に変更
- backlog/README.md を更新

### やらないこと
- GitHub Issues テンプレート（.github/ISSUE_TEMPLATE/）の作成（ワークフロー内で body を構成するため不要）
- 既存 entries/*.md のファイル名変更（ID はメタ情報として追加のみ）

## 外部リポジトリ

| リポジトリ名 | ローカルパス | 変更内容の概要 |
|-------------|------------|-------------|
| なし | - | - |

## 期待される効果

- BL-xxx の短い ID で施策を一意に指定可能になる
- ワークフロー実行時の入力ミスが減少する
- backlog → issue の転記が統一されたフォーマットで行われる

## リスク
| リスク | 影響度 | 対策 |
|--------|--------|------|
| CSV のカラム追加による既存ワークフローへの影響 | 低 | awk の列番号を調整する |

## 壁打ちの背景

ユーザーから「backlog に一意の ID を割り当てて指定したい」「backlog から issue への起票は backlog の内容をそのまま記載する方式としたい」という要望があった。ID 形式は BL-001 形式を選択。既存53件への遡及適用も含める。
