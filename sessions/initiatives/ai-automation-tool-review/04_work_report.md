## T-002: 自動化手段の調査

### Claude Code hooks の仕様概要

Claude Code hooks は以下のライフサイクルイベントで発火する:
- **PreToolUse**: ツール呼び出し前（バリデーション・ブロック用）
- **PostToolUse**: ツール呼び出し後（後処理・チェック用）
- **Notification**: 通知発生時
- **Stop**: セッション終了時（クリーンアップ用）
- **SessionStart**: セッション開始時（初期化用）

ハンドラー種別: `command`（シェルコマンド実行）

### 各定型作業の自動化手段

| # | 定型作業 | 自動化手段 | 実現可能性 | 備考 |
|---|---------|-----------|-----------|------|
| 1 | ブランチ整理 | **SessionStart hook** (command) | 高 | 7ステップをシェルスクリプト化し、SessionStart で自動実行。最も効果が高い |
| 2 | コミットメッセージ規約 | **PreToolUse hook** (Bash:git commit) | 高 | git commit 実行前にメッセージ形式をバリデーション。正規表現チェックで実装可能 |
| 3 | PR URL 出力形式 | **PostToolUse hook** | 低 | 出力テキストのチェックは hooks では困難。skills 定義での指示が現実的 |
| 4 | テンプレートコピー | **シェルスクリプト** | 中 | `cp -r _template/ <施策名>/` の1コマンド化。ただし施策名の入力が必要なため完全自動化は困難 |
| 5 | 壁打ちフェーズ | **skills 拡張** | 低 | 対話的プロセスのため自動化不向き。テンプレート化で効率化は可能 |
| 6 | 作業履歴記録 | **PostToolUse hook** + シェルスクリプト | 中 | タイムスタンプ自動付与は可能だが、内容の記述は手動が必要 |
| 7 | 知見記録 | **Stop hook** | 中 | セッション終了時に知見記録の有無をチェックし、未記載なら警告 |
| 8 | 課題起票・CSV転記 | **シェルスクリプト** | 中 | ISS-XXX の採番と CSV 追記をスクリプト化。テンプレート生成も自動化可能 |
| 9 | ゲート判定 | **skills 拡張** | 低 | 判断を伴うため自動化不向き。チェックリストの自動生成は可能 |
| 10 | 施策クローズ | **シェルスクリプト** | 高 | mv + CSV更新 + git add/commit をスクリプト化可能 |
| 11 | トリアージ走査 | **シェルスクリプト** | 中 | inbox/backlog のファイル数カウント・CSV 状態サマリーをスクリプト化可能 |
| 12 | ルール変更連動チェック | **Stop hook** | 中 | `.claude/rules/` の変更を検知し、`docs/workflow.md` の更新漏れを警告 |
| 13 | マネージャー共通ポリシー | **skills 拡張** | 低 | 対話的プロセスのため自動化不向き |
| 14 | 外部リポ整理 | **SessionStart hook** | 高 | ブランチ整理（#1）の一部として統合可能 |

## T-004: 優先度付きロードマップ

### Phase 1（即時実施可能・高効果）

#### 1. セッション開始時ブランチ整理の自動化（優先度スコア: 9.0）
- **手段**: SessionStart hook (command ハンドラー)
- **実装方針**: `.claude/rules/session-start-branch-cleanup.md` の7ステップをシェルスクリプト化し、`.claude/hooks/session-start-branch-cleanup.sh` として配置。SessionStart イベントで自動実行
- **含む範囲**: 外部リポ整理（#14、スコア6.0）も統合
- **想定工数**: 小（シェルスクリプト1ファイル + settings.json の hooks 設定）
- **リスク**: gh CLI のエラーハンドリング（ネットワーク障害時のフォールバック）

#### 2. コミットメッセージ規約の自動バリデーション（優先度スコア: 6.0）
- **手段**: PreToolUse hook (command ハンドラー、対象: Bash で git commit を含む場合)
- **実装方針**: コミットメッセージが `[session-type] category: summary` 形式に準拠しているかを正規表現でチェック。不準拠の場合はエラーを返しコミットをブロック
- **想定工数**: 小（バリデーションスクリプト1ファイル + hooks 設定）
- **リスク**: 偽陽性（正しいメッセージのブロック）への対処が必要

#### 3. 施策クローズの自動化（優先度スコア: 6.0）
- **手段**: シェルスクリプト（slash command として登録可能）
- **実装方針**: `施策名` を引数に取り、(1) `_archive/` への移動、(2) `backlog.csv` のステータス更新、(3) git add/commit を一括実行
- **想定工数**: 小（シェルスクリプト1ファイル）
- **リスク**: CSV のフォーマット崩れへの対処
