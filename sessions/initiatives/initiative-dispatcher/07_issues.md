### [2026-03-15] [評価] dispatch_inputs のシェルインジェクションリスク
- **種別**: セキュリティ課題
- **起票者**: L2（評価）
- **内容**: `initiative-dispatcher.yml` の Route command ステップで、ユーザーのコメント本文から抽出した `ARGS` 変数を `dispatch_inputs` の JSON 文字列に直接展開している（例: `"feedback\":\"${ARGS}\"`）。さらに Dispatch workflow ステップで `eval gh workflow run "$WORKFLOW" $input_flags` を使用している。`ARGS` に `"`, `\`, `$()`, バッククォート等が含まれる場合に JSON 破壊やシェルインジェクションが成立する可能性がある。
- **対応方針（案）**: (A) `jq` による JSON エスケープを使用する、(B) JSON 構築を廃止し各コマンド分岐で `-f` フラグを直接構築する方式に変更する。いずれの場合も `eval` の使用を廃止する。
- **影響範囲**: `initiative-dispatcher.yml` の Route command ステップ（行86-146）および Dispatch workflow ステップ（行148-163）
- **関連評価項目**: E-008
- **転記判断**: 要転記（施策をまたぐセキュリティパターン。今後のワークフロー開発でも同種のリスクが発生しうる） [転記済 ISS-059]
