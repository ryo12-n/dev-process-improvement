# TG-008チェック基準のルール変更連動更新

## メタ情報

| 項目 | 内容 |
|------|------|
| **優先度** | 🔴 高 |
| **ステータス** | 候補 |
| **対象リポジトリ** | dev-process-improvement |
| **起票日** | 2026-03-06 |

## 課題・背景

ルールファイル（rules/ 配下など）を変更した際に、TG-008 のチェック基準が連動して更新されていない。ルールの実態とチェック基準の間に乖離が生じると、トリアージ評価の精度が低下し、品質保証の信頼性が損なわれる。

## 期待効果

- テンプレート変更施策に「TG-008 基準の連動更新タスク」を標準タスクとして組み込むことで、ルール変更時の更新漏れを防止できる
- チェック基準とルールファイルの整合性が常に維持され、トリアージ評価の信頼性が向上する

## 補足・参考情報

inbox 原文からの施策化。ルールファイル変更時に TG-008 基準も連動更新すべきという知見に基づき、テンプレート変更施策に連動更新タスクを標準化する方針。
