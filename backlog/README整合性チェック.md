## 課題・背景

README.md は人間のエントリーポイントであり、リポジトリの全体像を正確に反映している必要がある。しかし、施策の積み重ねにより README の記載内容が実態と乖離している可能性がある。

### 確認済みの乖離例

- **sync セッションタイプの欠落**: `sync-manager` / `sync-worker` / `sync-evaluator` が追加されたが、README のセッション種別テーブル・ディレクトリ構成・AI向けスキル定義テーブルに反映されていない
- **`sessions/sync/` ディレクトリの欠落**: ディレクトリ構成図に `sessions/sync/` が含まれていない
- **`repo-sync-checklist` スキルの欠落**: `.claude/skills/` 配下のスキル一覧に含まれていない
- **triage テンプレートの乖離**: README では `02_work_log.md`, `03_report.md` の2ファイル構成だが、実態は `00〜04` + `workers/_template/` の構成に変更済み
- **backlog の構成**: README では `backlog/ideas.md` を参照しているが、実態は1施策1ファイル形式 + `backlog.csv`

## 期待効果
