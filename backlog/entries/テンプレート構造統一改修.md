## 課題・背景

session-lifecycle-policy §1.4 で基本モデル（multi-phase manager パターン）を定義したが、現状のテンプレートディレクトリ構造は以下の 3 点で基本モデルと乖離している:

1. **single-phase セッション（triage, meta, bm, auto）**: `workers/_template/` を使用しており、`phase-1-xxx/_template/` に統一されていない
2. **initiative**: フラット構造（ワーカー/評価ワーカーのファイルがルート直下）。マネージャー/ワーカー層の分離がない
3. **impl**: phase 内にファイル直置き（`_template/` sub-dir なし）。worker set ディスパッチパターンと不一致

## 期待効果

- 全セッションタイプのテンプレートが基本モデル（multi-phase manager パターン）に準拠し、構造的な一貫性が向上
- 新セッションタイプ追加時のテンプレート設計が容易になる
- SC-004/SC-009 チェックの例外ケースが減少

## 補足・参考情報

- 影響範囲が大きい（テンプレート変更 → エージェント定義の担当ファイルテーブル更新 → 既存セッションインスタンスとの互換性）
- 段階的に実施を推奨:
  1. single-phase の `workers/` → `phase-1-xxx/` 統一（影響小）
  2. initiative のマネージャー/ワーカー層分離（影響中）
  3. impl の `_template/` sub-dir 導入（影響大）
