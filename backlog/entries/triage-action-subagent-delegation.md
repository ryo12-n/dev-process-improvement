# トリアージアクション実施のサブエージェント委託

> **バックログ ID**: BL-049


## 概要

トリアージマネージャーの「アクション実施」フェーズ（inbox処理・backlog更新・CSV更新・README修正等）をサブエージェントに委託し、マネージャーのコンテキスト消費を削減する。

## 背景

トリアージ 20260312 で、アクション実施がマネージャー自身のコンテキストで約20回のツール呼出しを消費。特に backlog.csv 全文リライトと README.md の6箇所編集がコンテキストを圧迫した。アクションの8割は 03_report.md のテーブルから機械的に実行可能であり、サブエージェントへの委託が可能。

## 提案内容

1. `triage-action-worker` エージェント（または既存 triage-worker の拡張）を追加
2. 入力: 03_report.md の「backlog 追加・変更案」「削除・クローズ候補」「CRITICAL修正」テーブル
3. 出力: ファイル操作実施済み + 変更サマリ
4. マネージャーは差分確認 → コミット・PR のみ担当

## 期待効果

- マネージャーのコンテキスト消費を約15ツール呼出し分削減
- ゲート判定後の余力を PR レビュー品質に活用
- アクション実施の標準化（レポートテーブル駆動で再現性向上）

## 対象リポジトリ

dev-process-improvement

## 検討事項

- 既存の triage-worker を拡張するか、別エージェントにするか
- README.md 更新のように既存構造の理解が必要な作業の委託可否
- コミット・PR 作成はマネージャー残留が妥当（ブランチ名・メッセージの責務）
