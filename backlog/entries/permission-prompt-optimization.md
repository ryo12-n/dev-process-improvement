## 課題・背景

CLI 対話型環境で作業中、`.claude/settings.json` の allow リストに含まれていないコマンドを実行するたびに許可プロンプトが表示され、作業が中断される。allow リストは手動で事前設定する方式のため、実際に必要なコマンドを網羅するのが難しい。

## 期待効果

- セッション中に発生した許可待ちコマンドを作業ログに記録し、セッション終了時にまとめて allow リストへの追加可否を確認するフローを導入する
- 実際の利用パターンに基づいた allow リストの継続的な最適化が可能になる
- 許可プロンプトによる作業中断の段階的な削減

## 補足・参考情報

- `cli-permission-verification` 施策で、allow リストが CLI 対話型環境で許可プロンプトの抑制に有効であることを確認済み
- 実装案: L2-worker / L2-evaluator のエージェント定義、または rules/ に「許可待ち発生時の記録ルール」を追加し、セッション終了時に settings.json 更新を提案するフローを組み込む
- deny リストとの競合チェック（deny 対象コマンドを allow に追加しないようにする）も考慮が必要
