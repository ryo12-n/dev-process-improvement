## 凡例

| 項目 | 記法 | 説明 |
|------|------|------|
| **優先度** | 🔴 高 | 早期対応が必要。ブロッカーまたは高頻度問題、もしくは Claude Code の振る舞いを変える変更（セッションフロー・テンプレート・ルール等） |
| | 🟡 中 | 重要だが緊急ではない |
| | 🔵 低 | 余裕があれば対応 |

| | initiative 開始済 | `sessions/initiatives/` で施策進行中 |
| | 完了 | 施策クローズ済み |

## 課題・背景

- backlog.csv に一意の ID がなく、施策名（日本語テキスト）が実質的な識別子になっている
- ワークフロー（backlog-to-issue）で施策を指定する際、長い日本語名を正確に入力する必要がある
- backlog → issue 起票時にテンプレートがなく、転記内容がワークフローのシェルスクリプトにハードコードされている

## 期待効果

- BL-xxx の短い ID で施策を一意に指定可能になる
- ワークフロー実行時の入力ミスが減少する
- backlog → issue の転記が統一されたフォーマットで行われる

## 補足・参考情報

- 過去に initiative として計画済み（`sessions/initiatives/backlog-id-and-issue-template/`）だが実装未着手のためアーカイブ → バックログに戻した
- 既存53件への BL-001〜BL-053 採番は完了済み。残りは backlog/entries/*.md へのメタ情報追記とワークフロー更新
