## 課題・背景

`プロセス改善_課題管理.csv` は複数セッション（L1施策・トリアージ等）が並行して行を追加するため、マージ時にコンフリクトが頻発している。直近の例では ISS-059 の ID 衝突が発生し、リナンバー＋参照更新の手作業が必要になった。

CSV は行単位の追記が主な操作だが、git のマージは同一ファイル末尾への追記を競合として扱うため、並行作業と本質的に相性が悪い。

## 期待効果

- CSV マージコンフリクトの発生頻度をゼロまたは大幅に削減
- コンフリクト解消にかかる手作業（リナンバー・参照更新）の排除
- 並行セッション間での ID 衝突の防止

## 補足・参考情報

- 考えられるアプローチ例:
  - CSV を廃止し、個別ファイル（`issues/entries/ISS-XXX.md`）をメタデータの正の情報源とする方式への移行
  - CSV を自動生成（個別 .md ファイルから集約スクリプトで生成）する方式
  - ID 採番をセッション種別ごとにプレフィクス分離する方式
  - マージ前に CSV を自動ソート/再生成する git hook の導入
