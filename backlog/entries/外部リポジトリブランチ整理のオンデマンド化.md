# 外部リポジトリブランチ整理のオンデマンド化

|------|------|
| **優先度** | 🟡 中 |

| **起票日** | 2026-03-09 |

## 課題・背景

現在 `.claude/rules/session-start-branch-cleanup.md` のステップ6で、セッション開始時に**毎回**外部リポジトリのブランチ整理を実施している。しかし、外部リポジトリを操作しないセッション（例: dev-process-improvement 内で完結するトリアージや施策）でも整理が走るため、不要なオーバーヘッドが発生している。

具体的な問題:
- 外部リポジトリが存在しないリモート環境（SDK）ではパスが見つからず警告が出るだけで無駄
- セッション開始時のブランチ整理ステップが長くなり、本来の作業開始が遅れる
- 外部リポジトリの状態確認のために全施策の `00_proposal.md` を走査するコストがかかる

## 期待効果

- セッション開始時のオーバーヘッド削減（外部リポジトリ不要なセッションが大半）
- ブランチ整理ルールの責務分離（本体リポジトリの整理と外部リポジトリの整理を独立管理）
- 外部リポジトリ操作時のみ、適切なタイミングで整理を実行できる

## 実現案

1. `.claude/rules/session-start-branch-cleanup.md` のステップ6（外部リポジトリの整理）を抜き出し、独立したスキル（例: `.claude/skills/external-repo-cleanup/SKILL.md`）として定義する
2. セッション開始時のブランチ整理ルール（ステップ1〜5 + ステップ7）はそのまま維持する
3. 外部リポジトリを操作する施策の作業開始前に、マネージャーまたはワーカーが当該スキルを呼び出す
   - 呼び出しタイミング例: L1 が施策の `00_proposal.md` で外部リポジトリありと判断した時点
   - またはワーカーディスパッチ時に外部リポジトリ操作を含むタスクがある場合
