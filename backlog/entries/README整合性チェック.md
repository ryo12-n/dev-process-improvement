## 課題・背景

README.md は人間のエントリーポイントであり、リポジトリの全体像を正確に反映している必要がある。しかし、施策の積み重ねにより README の記載内容が実態と乖離している可能性がある。

### 確認済みの乖離例

- **sync セッションタイプの欠落**: `sync-manager` / `sync-worker` / `sync-evaluator` が追加されたが、README のセッション種別テーブル・ディレクトリ構成・AI向けスキル定義テーブルに反映されていない
- **`sessions/sync/` ディレクトリの欠落**: ディレクトリ構成図に `sessions/sync/` が含まれていない
- **`repo-sync-checklist` スキルの欠落**: `.claude/skills/` 配下のスキル一覧に含まれていない
- **triage テンプレートの乖離**: README では `02_work_log.md`, `03_report.md` の2ファイル構成だが、実態は `00〜04` + `workers/_template/` の構成に変更済み
- **backlog の構成**: README では `backlog/ideas.md` を参照しているが、実態は1施策1ファイル形式 + `backlog.csv`

## 期待効果

- 人間がリポジトリの全体像を正確に把握できるようになる
- 新しいセッションタイプ（sync）の存在と起動方法が README から分かるようになる
- 毎回のトリアージで README の乖離が自動検出され、放置されなくなる

## やること

本施策は2つの成果を求める。

### 1. 現時点の乖離の修正
README.md の記載内容を実態に合わせて修正する（上記「確認済みの乖離例」を解消する）。

### 2. トリアージへの README 整合性チェックのルール化
毎回のトリアージセッションで README と実態の整合性が継続的にチェックされるよう、以下をルール化する。

- `triage-manager/SKILL.md` または `triage-worker.md` の TG タスクとして「README 整合性チェック」を追加する
- チェック観点: ディレクトリ構成図・セッション種別テーブル・スキル定義テーブル・ドキュメントマップが実態と一致しているか
- 乖離を発見した場合のアクション（そのセッション内で修正 or 課題起票）を定義する

## 補足・参考情報

- README は CLAUDE.md のドキュメント階層方針により「エントリーポイントに徹する・150行以内」を目標としている
- `docs/workflow.md` の整合性チェックは既にトリアージの事前調査（00_pre_investigation.md）でルール化されている。README も同様の仕組みで継続的に維持する
