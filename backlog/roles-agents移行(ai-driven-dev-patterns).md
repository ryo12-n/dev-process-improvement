## 課題・背景

現在、ロール定義は `roles/` ディレクトリに Markdown ファイルとして配置されている。Claude Code は `.claude/agents/` ディレクトリでカスタムサブエージェントをネイティブにサポートしており、移行によりいくつかのメリットが得られる。

### 移行のメリット

- **独立コンテキスト**: agents/ のサブエージェントはメインセッションとは別のコンテキストウィンドウで動作し、メインコンテキストを汚染しない
- **ツール制限**: `tools:` フィールドで使用可能ツールを制限でき、ロールのスコープ逸脱を仕組みで防げる（例: reviewer には Read/Grep/Glob のみ）
- **skills プリロード**: `skills:` フィールドで必要なスキルだけ渡せる。メインセッションの skills を継承しない
- **Claude Code ネイティブ統合**: Agent ツールの subagent_type で直接指定可能

### 移行のデメリット・リスク

- **移行コスト**: 7つのロール定義ファイル全てのフォーマット変換（Markdown → YAML フロントマター付き agents 形式）が必要
- **dev_manager との連携変更**: dev_manager が roles/ を参照するオーケストレーションロジックの変更が必要
- **sessions/ ワークフローとの整合**: セッション管理との連携を再設計する必要がある
- **下位互換の断絶**: roles/ を参照している既存のスキル・ドキュメント全てを更新する必要がある

## 期待効果

- ロールのスコープ制御が仕組みレベルで強化される
- サブエージェントの独立コンテキストにより、メインセッションの精度が向上する
- Claude Code の標準機能に準拠した一貫性のある構成になる
