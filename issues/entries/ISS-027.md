## 課題詳細

Claude Code Web のクラウド環境はセッション作成時に紐づけたリポジトリのみ Git プロキシが認可する。フルネットワークアクセス設定でも別リポジトリへの git push は repository not authorized で拒否される。ネットワークアクセス設定（なし/信頼済み/フル/カスタム）は外部通信範囲の制御であり、Git プロキシのリポジトリ認可とは別レイヤー。リポジトリ分離を行うと分離先での作業に別セッションが必要となり、親リポのコンテキスト（CLAUDE.md・roles/・openspec/ 等）を直接参照できなくなる。

## 対応方針（案）

[案] ①モノレポ構成を維持し分離を見送る（現状の方針） ②Claude Code がマルチリポ対応した時点で再検討 ③複数リポへの同時操作が必要な場合はローカル環境で実施
