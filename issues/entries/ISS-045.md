## 課題詳細

L2-worker サブエージェントの実機検証（T-001）で、`.claude/settings.json` の `permissions.deny` に含まれるコマンド（`rm -rf` 等）が拒否されず正常実行された。同様に、`permissions.allow` に含まれないコマンド（`date`, `pwd`, `whoami`）も許可プロンプトなしで実行された。

これは Claude Agent SDK 経由のサブエージェント環境での結果であり、Claude Code CLI の対話型環境とは権限制御の挙動が異なることを示している。allow/deny リストは CLI の対話型セッションを前提とした仕組みであり、SDK/API 経由の実行では別の制御メカニズムが必要になる可能性がある。

## 対応方針（案）

[案] deny リストを「CLI 環境でのセーフガード」として位置づけ、SDK 環境では別途の安全策（サンドボックス、ファイルシステム制限等）が必要である旨を運用ガイドに明記する。deny リストのみに依存したセキュリティ設計は避けること。
