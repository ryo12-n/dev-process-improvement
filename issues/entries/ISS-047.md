## 課題詳細

エージェント定義ファイルの `tools` フィールド（例: `tools: ["Read", "Write", "Edit", "Glob", "Grep", "Bash"]`）で指定したツールリスト外のツールであっても、ToolSearch 経由で deferred tools として発見・ロード・実行が可能であることが CLI 環境での実機検証で判明した。

検証で利用可能だった tools リスト外のツール:
- WebSearch, WebFetch, NotebookEdit, TaskCreate/Update/List/Get, EnterWorktree, CronCreate/Delete/List

これは `tools` フィールドが「初期ロードされるツールセットの定義」であり、「利用可能なツールの制限」ではないことを意味する。

SDK 環境では tools フィールド自体が無視されていた（ISS-046）が、CLI 環境では初期ツールには適用されつつ deferred tools には適用されないという、より微妙な挙動差がある。

## 対応方針（案）

[案]
1. tools フィールドは意図の文書化として維持する（最小権限の原則の表明）
2. 完全なツール制限が必要な場合は `disallowedTools` フィールドの活用を検討する
3. `docs/subagent-permission-guide.md` に deferred tools の制限外であることを明記済み
