## 課題詳細

`.claude/agents/feature-builder.md` と `test-writer.md` にテスト実行コマンドが以下のように TODOのまま未設定：

```
<!-- TODO: 本リポジトリのCIスクリプトパスに修正 -->
```

TDD フロー（test_writer 先行 → feature-builder 実装）が設計上は定義されているが、実際のテスト実行コマンドが未設定のため TDD が機能しない状態。

**影響範囲**:
- feature-builder エージェント: 実装後のテスト実行ステップが機能しない
- test-writer エージェント: テストスクリプト作成後の検証ができない
- ハーネスエンジニアリング観点4（検証とガードレール）のスコア低下要因

## 対応方針（案）

[案] ai-driven-dev-patterns リポジトリのテストフレームワーク・CIスクリプトを確認し、以下を実施する：
1. 実際のテスト実行コマンドを調査する（package.json の scripts、Makefile 等）
2. `feature-builder.md` と `test-writer.md` の TODO をコメントを実際のコマンドに置き換える
3. CI連携が可能な場合は `.github/workflows/` への統合も検討する
