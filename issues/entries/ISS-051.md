## 課題詳細

施策「外部リポジトリ変更時のブランチ・PR運用ルール整備」で、外部リポジトリでの PR 操作に `gh -C <path>` 形式のコマンドが記載された。しかし、gh CLI v2.87.3 では `-C` フラグはサポートされておらず、`unknown shorthand flag: 'C' in -C` エラーが発生する。

`git -C <path>` は正常に動作するため、ブランチ確認・push・checkout には問題がない。影響は PR の確認・作成ステップ（`gh pr list` / `gh pr create`）のみ。

影響箇所:
1. `.claude/rules/session-start-branch-cleanup.md` ステップ6 — `gh -C <path> pr list` / `gh -C <path> pr create`
2. `.claude/skills/l1-manager/SKILL.md` クローズ手順ステップ7 — `gh -C <path> pr create`

## 対応方針（案）

[案] `gh -C <path>` の記載を `cd <path> && gh pr list/create` に変更する。`git -C` との混在になるが、`gh` 側に `-C` オプションがない以上フォールバックが必要。将来 gh CLI が `-C` をサポートした場合は `git -C` と統一できる。

## 補足・参照

- L2-worker の 04_work_report.md 参考情報で「`gh -C` の互換性要確認」と指摘されていたが、ルールファイルへの反映時に検証されなかった
- `git -C` はサポートされている: `git -C <path> branch --show-current` 等は問題なく動作する
- gh CLI バージョン: v2.87.3 (2026-02-23)
