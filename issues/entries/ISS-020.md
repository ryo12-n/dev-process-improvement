## 課題詳細

config.yaml の tasks ルールで「DB / API / フロントエンド / テスト のカテゴリに分けること」と指定されているが、CLI ツールや非Web系プロジェクトではこのカテゴリ分けが適合しない。

## 対応方針（案）

[案] config.yaml の tasks ルールを「プロジェクトタイプに応じた適切なカテゴリに分けること」のような柔軟な表現に変更するか、デフォルトカテゴリを参考例として示す形にする。
