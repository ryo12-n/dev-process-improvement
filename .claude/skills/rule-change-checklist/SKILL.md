# ルール変更時の連動更新チェックリスト

> **このファイルの位置づけ**
> ルール変更を含む施策（`.claude/skills/` や `.claude/rules/` 配下のファイルを変更する施策）で、
> 連動更新の漏れを防ぐための標準チェックリスト。
> L1 マネージャーが `02_tasks.md` にタスクを含める際、または L2 ワーカーが作業完了前に確認する際に使用する。

---

## 標準タスクパターン: 調査 → 文書化 → ルール反映 → workflow 更新

ルール変更を含む施策では、以下の定型パターンに従う。

```
1. 調査        — 現状の参照関係・影響範囲を把握する
2. 文書化      — 設計・ガイドライン文書を作成する（必要な場合）
3. ルール反映  — .claude/skills/ や .claude/rules/ のファイルを変更する
4. 連動更新    — 以下のチェックリストに従い、関連ファイルを更新する
```

---

## チェックリスト

### 1. 関連ファイル一覧の確認

変更したファイルに「関連ファイル一覧」セクションがある場合、そのテーブルに記載された全ファイルについて連動更新の要否を確認する。

- [ ] 変更ファイルの「関連ファイル一覧」セクションを確認した
- [ ] 一覧に記載された各ファイルについて、連動更新の要否を判断した
- [ ] 連動更新が必要なファイルを更新した

### 2. docs/workflow.md の更新

`.claude/skills/` 配下のスキル・エージェント定義が正の情報源（Source of Truth）であり、`docs/workflow.md` はその人間向け可視化である。ルールファイルを変更した場合は、必ず `docs/workflow.md` も合わせて更新する。

- [ ] 変更内容が `docs/workflow.md` のどのセクションに影響するかを特定した
- [ ] `docs/workflow.md` の該当セクションを更新した
- [ ] 更新後の `docs/workflow.md` が変更後のルールと整合していることを確認した

### 3. TG-008 基準の連動更新

ルール変更が `.claude/skills/triage-standard-policy/SKILL.md`（トリアージ標準ポリシー）のチェック基準に影響する場合、基準文書も更新する。

- [ ] 変更内容が以下のいずれかに該当するかを確認した:
  - ライフサイクルステージの追加・変更・削除
  - ペアリング対称性要件の変更
  - 課題起票・ルーティング手段の変更
  - 停止ルール・スコープの変更
- [ ] 該当する場合、`triage-standard-policy/SKILL.md` のチェック基準を更新した
- [ ] 適用マトリクス（セクション 1.2）に変更が必要な場合は更新した

### 4. テンプレートの連動更新

変更がテンプレートファイルの構成に影響する場合、テンプレートも更新する。

- [ ] イニシアティブ系: `sessions/initiatives/_template/` 配下のテンプレートに影響がないか確認した
- [ ] トリアージ系: `sessions/triage/_template/` 配下のテンプレートに影響がないか確認した

### 5. コミットメッセージ規約の確認

新しいセッション種別を追加した場合、コミットメッセージ規約にも反映する。

- [ ] `.claude/rules/commit-message.md` に新セッション種別の追加が必要か確認した

## 使い方

### L1 マネージャー（02_tasks.md 作成時）

ルール変更を含む施策では、実施タスクの最後に以下の固定タスクを追加する:

```markdown
| T-XXX | ルール変更連動更新チェックリスト（rule-change-checklist スキル）を実施する | チェックリストの全項目が確認済み | 🔴 | ⬜ |
```

### L2 ワーカー（作業完了前）

ルールファイルを変更した後、コミット前にチェックリストの各項目を確認し、`03_work_log.md` に確認結果を記録する。
