# セッション開始時のブランチ整理ルール

## 適用対象

- L1 マネージャーセッション（`.claude/skills/l1-manager/SKILL.md`）
- トリアージマネージャーセッション（`.claude/skills/triage-manager/SKILL.md`）

**L2 サブエージェント（worker / evaluator）は対象外**。L2 は L1 からサブエージェントとして起動されるため、ブランチ管理は L1 の責務である。

## 実行タイミング

セッション開始時、スキル定義の作業フロー（入力解釈・テンプレートコピー等）に入る**前**に実行する。

## フロー

以下の 6 ステップを順に実行する。

### ステップ 1: 現在ブランチの確認

```bash
git branch --show-current
```

- 結果が `main` の場合 → ステップ 5 の `git pull` のみ実行し、ステップ 6 へ進む
- 結果が `main` 以外の場合 → ステップ 2 へ進む

### ステップ 2: 未プッシュコミットの確認

```bash
git log origin/<現在のブランチ>..HEAD --oneline
```

- 出力がある場合（未プッシュコミットあり）→ ステップ 3 へ
- 出力がない場合（全コミットプッシュ済み）→ ステップ 4 へ
- リモートブランチが存在しない場合 → ステップ 3 へ（初回プッシュが必要）

### ステップ 3: 未プッシュコミットのプッシュ

```bash
git push -u origin <現在のブランチ>
```

- プッシュ成功 → ステップ 4 へ
- プッシュ失敗 → エラーを報告し、ユーザーに判断を仰ぐ（セッションを停止する）

### ステップ 4: PR の確認・作成

```bash
gh pr list --head <現在のブランチ> --base main --state open
```

- PR が存在する場合 → PR の URL を確認メッセージとして表示し、ステップ 5 へ
- PR が存在しない場合 → 以下で PR を作成する

```bash
gh pr create --base main --head <現在のブランチ> --title "<ブランチ名の要約>" --body "前セッションからの継続作業。セッション開始時のブランチ整理で自動作成。"
```

- PR 作成成功 → 作成した PR の URL を表示し、ステップ 5 へ
- PR 作成失敗 → エラーを報告し、ユーザーに判断を仰ぐ

### ステップ 5: main ブランチへの切り替え・最新化

```bash
git checkout main && git pull
```

- 成功 → ステップ 6 へ
- 失敗 → エラーを報告し、ユーザーに判断を仰ぐ

### ステップ 6: 通常のセッションフローへ

ブランチ整理が完了した旨をユーザーに報告し、スキル定義の通常の作業フロー（入力解釈・テンプレートコピー等）に進む。

## 注意事項

- **マージは行わない**: PR の作成までが本フローの責務。マージはユーザーの判断に委ねる。
- **PR 作成済みならブロックしない**: 前セッションの PR がマージ待ちでも、main に切り替えて新しい作業を開始してよい。
- **同じ施策の続きでも整理する**: 前セッションと同じブランチにいても、一旦 main に戻してから再度ブランチに入る（壁打ちフェーズで施策を確認した後にブランチ切り替えを行う）。
