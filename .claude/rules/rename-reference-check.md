# リネーム・移動時の参照更新チェックルール

ファイルやディレクトリのリネーム・移動・廃止（DELETE）を行う場合、以下の手順で参照更新漏れを防止する。

## 手順

### 1. リネーム前: 旧名称の全文検索

リネーム・移動・廃止を実行する **前** に、旧名称で `Grep` 全文検索を実行し、参照箇所を把握する。

```bash
# 例: triage-standard-policy を session-lifecycle-policy にリネームする場合
Grep pattern="triage-standard-policy"
```

### 2. 検出箇所の一括更新

検出された **すべての箇所** を新名称に更新する。ファイル廃止（DELETE）の場合は、参照箇所を削除するか適切な代替先に更新する。

### 3. 更新後: 残存参照ゼロの確認

全箇所の更新が完了したら、旧名称で再度 `Grep` を実行し、残存参照がゼロであることを確認する。

```bash
# 残存参照がないことを確認
Grep pattern="triage-standard-policy"
# → "No matches found" であること
```

## 対象範囲

以下のディレクトリを走査対象とする。

- `.claude/skills/`
- `.claude/rules/`
- `docs/`
- `sessions/*/_template/`
- `backlog/`
- `templates/`

> **注意**: `session-consistency-reference.md` 等のリファレンスデータ（過去のスキャン結果を記録したもの）は、旧名称が事実の記録として残る場合があるため、走査結果から除外して判断してよい。

## ファイル廃止（DELETE）時の適用

ファイルを廃止する場合も同様に、廃止するファイル名・パスで全文検索を実行し、残存参照がないことを確認する。参照が残っている場合は、参照箇所を削除するか代替先に更新してから廃止する。

## rule-change-checklist との関係

本ルールは `rule-change-checklist` スキル（§4「パス変更・リネームの走査対象網羅性」）を補完する。`rule-change-checklist` はルール変更施策全体のチェックリストであり、本ルールはリネーム・移動・廃止という特定操作に特化した手順を定義する。
