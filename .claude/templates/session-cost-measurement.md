# セッションコスト計測テンプレート

## A. 概要・使い方

### 目的

施策のセッションコスト（トークン消費量・所要時間・リワーク率等）を定量的に計測・記録するための標準テンプレート。全施策に対して `09_cost_record.md` として適用する。

### 適用場面

- L1 が新施策を開始する際、`_template/09_cost_record.md` を施策ディレクトリにコピーする
- 本テンプレートの Section B〜C が `_template/09_cost_record.md` の設計基盤となる
- Section D（料金リファレンス）は `09_cost_record.md` から参照する（重複を避けるため本テンプレートに一元化）

### 計測方式

**前方計測**: 施策開始時に L1 が `09_cost_record.md` と `02b_dispatch_log.md` を初期化し、L1 が各ディスパッチ完了時にコストデータを記録する。

**データ収集ポイント**:

| タイミング | 収集者 | 収集内容 |
|-----------|--------|---------|
| ディスパッチ完了時 | L1（マネージャー） | `02b_dispatch_log.md` にコストデータ行（input_tokens/output_tokens）を記録 |
| Phase チェック時 | L1（マネージャー） | `09_cost_record.md` にフェーズコスト小計を記録 |
| ゲート判定時 | L1（マネージャー） | `08_gate_review.md` に施策全体コスト集計を転記、ゲート結果・リワーク有無を記録 |

**02b_dispatch_log.md へのコストデータ記録形式**:

L1 は各ディスパッチ完了エントリに以下の1行を含める。

```
**コストデータ**: input_tokens=_____ output_tokens=_____
```

### メトリクス定義

| メトリクス | 定義 | 単位 |
|-----------|------|------|
| 経過時間 | セッション開始〜終了のウォールクロック時間 | 分 |
| 入力トークン | セッション中に消費した入力トークン数 | tokens |
| 出力トークン | セッション中に消費した出力トークン数 | tokens |
| コスト | 料金リファレンス（Section D）に基づく算出コスト | USD |
| サブエージェント発行数 | L1→L2 のディスパッチ回数（L1セッションのみ） | 回 |
| リワーク | ゲート判定 Reject により再実施が発生したかどうか | Yes/No |
| リワーク率 | リワーク発生セッション数 / 全セッション数 | % |
| タスクあたりコスト | 総コスト / タスク数 | USD/task |

## D. 料金リファレンス

### モデル別単価

| モデル | 入力 (per 1M tokens) | 出力 (per 1M tokens) |
|--------|---------------------|---------------------|
| Claude Sonnet 4 | $3.00 | $15.00 |
| Claude Opus 4 | $15.00 | $75.00 |

### 算出式

```
コスト (USD) = (入力トークン × 入力単価 + 出力トークン × 出力単価) / 1,000,000
```

### 注意事項

- 料金は変更される可能性がある。計測時点の料金を使用し、キャンペーンファイルに使用した料金テーブルを記録すること
- トークン数が取得できない場合は `N/A` と記録し、経過時間をフォールバック指標として使用する
- Extended thinking のトークンは入力トークンに含まれる（モデルにより異なる場合がある）
