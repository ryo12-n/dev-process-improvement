# 費用対効果評価フレームワーク（4軸モデル）

## 概要

定型作業や改善候補の費用対効果を定量的に評価するための汎用フレームワーク。
3つの独立変数（頻度・手動コスト・実装難易度）から優先度スコア（算出結果）を導出する **4軸モデル** を採用する。

### 4軸の構成

| 種別 | 軸 | 説明 |
|------|-----|------|
| 独立変数 | 頻度 | 対象作業がどの程度の頻度で発生するか |
| 独立変数 | 手動コスト | 手動で実施した場合のコスト（時間・ステップ数・ミスリスク） |
| 独立変数 | 実装難易度 | 改善・自動化を実装する際の技術的な難しさ |
| 算出結果 | 優先度スコア | 3つの独立変数から算出される総合優先度 |

---

## 評価軸定義

### 頻度

| レベル | 値 | 説明 |
|--------|-----|------|
| 高 | 3 | 毎セッション（毎回発生する作業） |
| 中 | 2 | 毎施策（施策ごとに発生する作業） |
| 低 | 1 | 不定期（特定条件でのみ発生する作業） |

### 手動コスト

| レベル | 値 | 説明 |
|--------|-----|------|
| 高 | 3 | 5分以上 / 複数ステップ / ミスしやすい |
| 中 | 2 | 2〜5分 / 中程度のステップ数 |
| 低 | 1 | 1分未満 / 単純な操作 |

### 実装難易度

| レベル | 値 | 説明 |
|--------|-----|------|
| 高 | 3 | セマンティック判断・複雑ロジックが必要 |
| 中 | 2 | 中程度のスクリプト・設定変更 |
| 低 | 1 | 単純なコマンド・設定変更 |

---

## 算出式

```
優先度スコア = (頻度 x 手動コスト) / 実装難易度
```

- スコアが **高いほど優先度が高い**（費用対効果が大きい）
- 頻度と手動コストが高く、実装難易度が低い項目が最も高スコアとなる

## 評価テーブルテンプレート

以下のテーブルをコピーして使用する。列はドメインに応じてカスタマイズ可能（カスタマイズ指針を参照）。

```markdown
| # | 対象項目 | 頻度 | 手動コスト | 実装難易度 | 優先度スコア | 順位 |
|---|---------|------|-----------|-----------|------------|------|
| 1 | [項目名] | X | X | X | X.X | X |
```

---

## カスタマイズ指針

本テンプレートは汎用フレームワークとして設計されている。ドメイン固有の評価に適用する際は、以下の方針でカスタマイズする。

### カラムの追加

評価テーブルに **ドメイン固有のカラム** を追加できる。基底の7カラム（#、対象項目、頻度、手動コスト、実装難易度、優先度スコア、順位）は変更せず、追加カラムを挿入する。

例: 自動化候補の評価では「自動化手段」カラムを追加する

```markdown
| # | 対象項目 | 頻度 | 手動コスト | 実装難易度 | 自動化手段 | 優先度スコア | 順位 |
```

### レベル定義の調整

各軸のレベル説明はドメインに応じて調整できる。値（1〜3）と算出式は変更しない。

例: 頻度のレベルを「毎日=3 / 毎週=2 / 毎月=1」に変更する

### 算出式の拡張

基本算出式を維持しつつ、重み付けを追加できる。

例: `優先度スコア = (頻度 x 手動コスト x 重み) / 実装難易度`
